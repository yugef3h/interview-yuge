# 快手 AI 应用开发 / AI Agent 二面面试题
面试时间：2026年5月5日

---

## 一、实习介绍（20 min）
- 从业务角度分析实习经历，包括接到需求后如何处理、如何设计。
- 实习工作的衡量指标，包括冷启动 0-1 阶段的指标，以及上线后如何通过用户反馈构筑数据飞轮。
- 了解哪些方法可以根据用户反馈全面、深入地挖掘用户真实痛点，并指导开发工作？实习中是否有相关经历？若有，请结合经历说明。

## 二、Harness Engineering 相关
- 什么是 Harness Engineering？你对它的理解是什么？
- 结合你的实习经历，说明如果要把 Harness 工程实践结合进项目，你会怎么做？

## 三、AI Agent Skill 开发
- 如果让你写 Agent 的 Skill（技能），你会怎么写？需要注意什么？

## 四、AI Agent 发展趋势
- 基于你对 Agent 的理解，你认为未来 Agent 的发展会如何演进、有哪些变化？

## 五、AI 对程序员的影响
- 你认为 AI 的出现，会对程序员的工作造成什么影响？

## 六、快手 AgentOS 底层投入策略
- 快手有很多 AgentOS 团队和 AI 应用开发团队，会做大量基础底层能力的建设。未来我们是否还需要花重资产去投入？可以从模型、框架、Skills 等方面展开说明，哪些方面需要重投入，哪些不需要？

## 八股文 / 基础技术题
1. MySQL 存储引擎有哪些？常用引擎的区别是什么？
2. MySQL 索引的底层结构是什么？为什么使用 B+ 树而不是 B 树或哈希表？
3. Redis 为什么快？
4. 针对 Redis 缓存一致性问题，有哪些解决方案？
5. 介绍一下 HTTPS 的握手过程。
6. 算法题：反转链表。
