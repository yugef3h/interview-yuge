## 二、字节/阿里/腾讯 真实高频题（社招2年+）

### 1）大模型基础（全栈转AI必问，不背定义要讲工程取舍）
1. Transformer 自注意力的计算复杂度？如何降低？（字节/阿里必问）
2. KV Cache 原理？推理时显存占用怎么算？长文本有哪些瓶颈与优化？（字节推理岗高频）
3. GQA/MQA 与 MHA 的区别？为什么能加速？KV 共享的取舍？（阿里/腾讯）
4. Decoder-only 对比 Encoder-Decoder 的工程优势？为什么现在大模型都用 Decoder-only？
5. 什么是 FlashAttention？核心优化点？为什么能省显存/提速？（字节AML/阿里达摩院）

### 2）RAG 全链路（2025–2026 全栈AI岗最高频）
1. RAG 完整流程？每一步的常见问题与优化手段？（被问5次以上）
2. Chunk size/overlap 怎么选？依据什么？如何做语义分块？
3. Embedding 模型选型（BGE/Text-embedding/商用）的取舍？中文场景注意什么？
4. 向量库选型（FAISS/Milvus/Weaviate/PGVector）对比？高并发/海量数据下怎么选？
5. 检索召回不准怎么办？Query 改写、多路召回、Rerank 怎么做？（阿里必问）
6. RAG 与 Fine-tuning 的适用场景？什么时候用哪个？怎么结合？

### 3）Agent / 工具调用 / MCP（2026 大厂核心）
1. Agent 架构设计：ReAct、Plan-and-Execute、CoT 的区别与落地场景？
2. Function Calling 原理？如何设计可靠的工具调用（参数校验、异常处理、重试）？
3. 多 Agent 协作：通信协议、任务拆分、记忆共享、冲突解决？（字节）
4. MCP 协议是什么？解决什么问题？和传统工具调用的差异？
5. Agent 记忆机制：短期/长期记忆怎么设计？如何做记忆压缩与检索？

### 4）推理优化（全栈转AI核心，资深必深挖）
1. 大模型推理性能指标：TPS、首包 latency、显存占用、吞吐量，怎么优化？（字节/阿里高频）
2. 量化技术：INT8/INT4/AWQ/GPTQ 原理？精度损失与加速比怎么权衡？
3. 动态批处理（Dynamic Batching）原理？如何处理不同长度请求？有哪些坑？
4. 推理服务化架构：如何做高可用、限流、熔断、灰度、多模型部署？
5. 端侧部署：ONNX/TensorRT/OpenVINO 优化？CPU/GPU 推理差异？

### 5）分布式训练/微调（资深岗分水岭）
1. 数据并行、张量并行、流水线并行的原理、适用场景、通信开销对比？（字节/阿里100%必考）
2. ZeRO 优化（stage1–3）原理？如何减少显存占用？通信代价？
3. LoRA 微调原理？对比全量微调的优势与局限？适用场景？
4. 分布式训练通信：NCCL、Gloo 区别？多机多卡如何做通信优化？

### 6）系统设计（全栈强项，面试官最爱）
1. 设计一个企业级 RAG 系统，支持千万级文档、高并发、低延迟、可扩展。（阿里/腾讯）
2. 设计一个大模型推理服务平台，支持多模型、动态扩缩容、灰度发布、监控告警。（字节）
3. 设计一个 Agent 任务编排系统，支持复杂任务拆分、工具调用、结果校验、失败重试。
4. 设计金融/医疗场景的私有化大模型部署架构，强调安全合规、数据不出域、性能优化。

### 7）线上故障与调优（资深必问，全栈优势）
1. 大模型推理服务上线后，首包延迟高、TPS 上不去，全链路排查思路？（字节/阿里高频）
2. RAG 系统检索结果质量差、用户投诉，如何定位问题并优化？
3. 分布式训练时 GPU 利用率低、训练慢，排查方向？
4. 模型上线后出现“幻觉”严重、回答不稳定，如何解决？

### 8）手写/工程题（社招常见，不考算法套路考AI工程）
1. 手写一个简单的 Transformer 多头注意力（PyTorch），重点处理数值稳定性。
2. 手写 KV Cache 的增量解码逻辑，说明显存管理。
3. 实现一个 RAG 的检索+重排流程（伪代码/简化代码）。
4. 用 Python 写一个异步并发的 LLM 调用客户端，支持限流、超时、重试。

## 五、全栈转AI 面试核心建议（针对你背景）
- **扬长避短**：重点讲**服务化架构、高并发、故障排查、性能调优**，这是纯算法岗短板。
- **项目深挖**：准备1–2个完整AI项目（如RAG平台、Agent系统、推理服务），能讲清**技术选型、取舍、踩坑、优化效果（量化）**。
- **原理+工程结合**：不要只背Transformer公式，要讲**为什么这么设计、工程上怎么实现、有什么坑**。
