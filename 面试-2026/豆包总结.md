# 一、月薪40K AI应用/AI全栈岗（社招为主）能力门槛定位（PS：降低预期，豆包的不一定准！！！！！）
结合国内一线/大厂薪资水平，**40K 月薪普遍对应：中高级 AI 应用工程师、AI Agent 开发工程师、大模型应用架构师**（3~5年相关经验），分**基础底线、核心能力、高阶加分**三层界定掌握程度，同时区分「纯应用开发」和「偏架构/落地」两个方向。

> 适用岗位：AI应用开发、AI Agent开发、大模型工程化、RAG全栈、LLM 业务落地、AI 全栈工程师

## （一）整体能力层级标准（对标40K）
### 1. 底线能力（达不到直接卡薪资＜30K）
1. **大模型基础**
   吃透 LLM 运行机制、Token、上下文窗口、Prompt 工程、基础采样参数（Temperature/TopP）；能独立写业务 Prompt、区分零样本/少样本/思维链。
2. **RAG 全链路**
   独立搭建传统 RAG：文档解析→切片→向量化→向量库部署→检索→召回+重排→LLM 生成；能排查**召回差、幻觉、上下文溢出**等常规问题。
3. **基础工具调用 & 简单 Agent**
   理解 `ReAct` 范式，能基于主流框架（LangChain/LlamaIndex）实现单 Agent、简单工具调用（API/代码执行/数据库查询）。
4. **工程基础**
   Python 高阶使用、接口开发（FastAPI）、基础容器化（Docker）、简单日志/异常处理、接口联调。
5. **问题排查**
   能独立定位：调用超时、Token 超限、检索不准、输出格式错乱等线上常见问题。

### 2. 核心能力（达到即可稳拿 35K~40K，大厂标配）
这是**薪资分水岭**，也是连环追问的核心区，要求**懂原理、能落地、会调优、能扛线上问题**，不再是只会调用框架 API 的“调包侠”。
1. **AI Agent 体系化能力**
   吃透 Agent 五大核心模块（感知/记忆/规划/工具/反思）、主流规划范式（ReAct/Plan&Execute/分层规划/ToT）；能**自研轻量 Agent 框架**，而非依赖 LangChain 堆砌。
   精通**三层记忆架构**：短期上下文、长期向量记忆、任务状态记忆；解决长任务漂移、死循环、断点续跑。
2. **RAG 深度优化 & 进阶方案**
   掌握分块策略优化、混合检索（关键词+语义）、多级重排、Query 改写/拓展/拆解；懂 **Graph RAG、自适应 RAG、增量 RAG**；能针对业务做 RAG 效果评测、指标体系搭建。
3. **大规模工具调度 & 工程稳定性**
   百级工具场景下的工具分类、向量预筛选、Schema 强校验、全链路异常重试/降级/熔断；懂沙箱隔离、权限管控。
4. **多智能体（Multi-Agent）实战**
   落地过 **Manager-Worker、流水线、辩论型** 多 Agent；设计角色分工、通信协议、任务调度、异常流转；能评估多 Agent 成本与性能。
5. **大模型工程化 & 部署**
   理解模型调用链路、流式输出、并发控制、限流、负载均衡；懂模型部署形态（API 服务、本地私有化、端云协同）；会做 Token 成本优化、QPS 压测、链路监控。
6. **幻觉治理 & 全链路质量管控**
   不再只说“用 RAG”，能设计**前置约束、过程校验、后置核查、Bad Case 回流**整套幻觉治理方案；有线上幻觉率优化实战经验。
7. **安全合规**
   懂提示注入防御、数据脱敏、最小权限原则、操作审计、隐私合规，能给业务输出安全方案。
8. **架构设计能力**
   能从 0~1 设计垂直场景 AI 应用架构（客服/数据分析/代码助手/运营助手等），拆分模块、做技术选型、评估优劣。

### 3. 高阶加分能力（40K 往上冲 45K+，技术专家方向）
1. 框架源码阅读：LangChain/LlamaIndex 核心流程、自定义组件开发；
2. 模型微调入门：SFT、数据构造、LoRA 轻量化微调场景选型；
3. MLOps 链路：数据集管理、模型版本、A/B 测试、灰度发布、全链路可观测；
4. 高并发/分布式 AI 应用：分布式向量库、分布式 Agent 调度、海量用户场景架构；
5. 前沿落地：多模态 Agent、端侧大模型应用、Agent 标准化协议探索。

# 二、面试官视角：40K 级别「夺命连环问」全套题库 + 追问逻辑 + 考点解析
整体追问逻辑：**基础概念 → 实操落地 → 痛点优化 → 架构设计 → 深层原理 → 线上问题 → 方案选型 → 行业思考**，层层递进，一旦回答浅层，立刻深挖细节。

分为 6 大模块，完全复刻大厂连环拷问节奏，每一组问题标注**追问链、考察点、踩坑点、高分回答方向**。

## 模块一：基础认知（开胃题，筛掉纯新手，1~2轮追问）
### 连环提问链
1. 问：你日常做 AI 应用开发，主要落地哪些场景？用到了哪些技术栈？
   - 追问1：你怎么理解 LLM、RAG、AI Agent 三者的关系？能不能用架构视角区分？
   - 追问2：很多人说 Agent = LLM + 工具调用，你认同吗？为什么？
   - 追问3：如果只给你原生大模型 API，不使用任何框架，你怎么手写一个最简单的 Agent？

### 考察目的
- 筛选认知层次：是停留在概念背诵，还是**从执行链路理解本质**；
- 区分“会用框架”和“懂底层逻辑”，40K 坚决淘汰只会套组件的人。

### 踩坑点
- 只背定义，不说链路；
- 认为 Agent 就是简单叠加工具；
- 手写原生 Agent 答不出核心流程。

### 高分应答核心
1. 三者是**组件与系统**关系：LLM 是推理核心，RAG 是记忆增强组件，Agent 是包含感知/记忆/规划/工具/反思的完整闭环系统；
2. 否定“LLM+工具=Agent”：缺少**任务规划、状态记忆、反思纠错**，只能叫插件调用，不是智能体；
3. 原生简易 Agent 流程：接收输入 → Prompt 定义规则+工具描述 → LLM 推理判断是否调用工具 → 解析工具参数 → 执行工具 → 结果回传给 LLM → 循环直到任务结束。

## 模块二：RAG 全链路（重中之重，最长追问链，大厂必深挖 5~8 轮）
RAG 是 AI 应用落地最通用场景，也是**区分初级/中级/高级工程师的核心**。

### 完整夺命追问链
1. 问：介绍一下你做的 RAG 项目，整体流程是什么？
   - 追问1：文档切片你用什么策略？固定切片、语义切片、层级切片分别怎么选？各自优缺点？
   - 追问2：切片大小怎么定？依据是什么？切片重叠（Overlap）为什么要设置？重叠多少合适？
   - 追问3：向量化用什么模型？开源/商用怎么选型？不同 embedding 模型对召回效果影响有多大？
   - 追问4：向量库用过哪些？FAISS、Milvus、Chroma、ES 分别适用什么场景？高并发/海量文档下怎么选型？
   - 追问5：检索阶段，为什么只用语义检索经常效果差？你做过哪些检索优化？
   - 追问6：什么是混合检索（关键词+语义）？怎么实现？重排（Rerank）在链路里起什么作用？什么时候必须加？
   - 追问7：线上 RAG 最头疼的问题是幻觉和召回不准，你分别做过哪些优化方案？
   - 追问8：如果文档是几十 GB 的海量 PDF/知识库，普通 RAG 扛不住，你有什么进阶方案？（引申 Map-Reduce / Graph RAG）
   - 追问9：怎么量化评估一套 RAG 系统好坏？有哪些核心指标？线上怎么持续监控？

### 考察目的
- 从“会搭流程”到**懂调优、懂选型、懂评测、懂海量场景架构**；
- 40K 要求：不仅能搭 RAG，还要能**针对业务问题给出定制化优化方案**。

### 高频踩坑点
1. 切片只会固定长度，说不出语义切片、层级切片的适用场景；
2. 不知道切片重叠的意义，凭感觉设置数值；
3. 只用语义检索，不了解混合检索、重排；
4. 解决幻觉只会说“多检索内容”，无全链路方案；
5. 无法给出 RAG 量化指标（召回率、精准率、答案相关性、幻觉率等）。

### 核心高分要点（精简）
1. **切片选型**
   - 固定切片：简单通用，适合纯文本、格式规整文档；缺点易割裂语义；
   - 语义切片：按句子/段落语义分割，保留完整性，适合文章、手册；
   - 层级切片：标题+正文绑定，适合结构化文档（帮助中心、知识库）。
   - 重叠：避免语义截断，常规 10%~20% 切片长度。
2. **检索优化**
   基础：Query 改写、Query 拓展、历史上下文融合；进阶：关键词检索（ES）+语义检索混合，搭配 Rerank 模型精排。
3. **海量文档方案**
   长文档：Map-Reduce 分块摘要；关联知识场景：Graph RAG 挖掘实体与关系；更新频繁场景：增量 RAG。
4. **评测指标**
   检索层：召回率、精确率；生成层：答案相关性、事实正确率、幻觉率、上下文贴合度；线上监控：QPS、延迟、检索耗时、Bad Case 数量。

## 模块三：AI Agent 核心（薪资核心分水岭，连环追问 6~10 轮，40K 必考）
### 完整夺命追问链
1. 问：你落地过哪些 AI Agent 场景？讲一个最复杂的项目架构。
   - 追问1：Agent 标准五大模块你怎么理解？你项目里每个模块具体怎么实现？
   - 追问2：业内说的**三层记忆架构**（短期/长期/工作记忆），分别解决什么问题？你怎么落地？
   - 追问3：主流规划范式 ReAct、Plan-and-Execute、分层规划、ToT，你都用过吗？场景怎么选型？
   - 追问4：线上长任务 Agent 经常出现**步骤漂移、忘记目标、死循环**，根因是什么？你的解决方案？
   - 追问5：工具很多（几十上百个）时，工具调用准确率暴跌，你怎么优化？
   - 追问6：什么是反思（Reflection）机制？你在项目里怎么设计反思流程来降幻觉、纠错？
   - 追问7：如果让你抛弃 LangChain 等框架，自研一套轻量 Agent 调度引擎，核心要实现哪些能力？
   - 追问8：Agent 执行链路很长，怎么做全链路监控、日志、问题定位？

### 考察目的
- 区分「调用框架组件」和「理解架构本质、能自研、能解决线上顽疾」；
- 30K 会用框架即可，**40K 必须懂模块设计、问题根因、自研思路**。

### 高频踩坑点
1. 分不清三种记忆的作用，把工作记忆等同于上下文；
2. 规划范式只会名词，说不出选型逻辑和优缺点；
3. 解决死循环只知道“加最大步数”，无状态机、异常检测、目标锚定等体系方案；
4. 工具优化只知道写详细描述，不懂向量预筛选、分类、Schema 校验。

### 核心高分要点
1. **三层记忆**
   - 短期记忆：对话上下文，保证多轮连贯，受窗口限制；
   - 长期记忆：向量库存储历史经验/知识库，RAG 调取，突破窗口；
   - 工作记忆：**任务状态快照**，记录步骤、进度、中间结果，实现断点续跑、状态管控（工业级核心）。
2. **长任务问题解决方案**
   - 漂移/忘目标：每轮 Prompt 强制锚定原始目标 + 里程碑阶段性校验 + 上下文摘要压缩；
   - 死循环：状态机记录执行轨迹、设置重复调用检测、区分异常类型做重试/降级、全局步数限制。
3. **大规模工具优化**
   工具分层分类 → 工具描述向量化预筛选 TopN → JSON Schema 强格式校验 → 少样本调用示例 → 调用异常分类处理。
4. **反思机制**
   分为**结果反思**（校验答案是否合规、有无幻觉）、**过程反思**（复盘执行步骤、定位失败原因），反思结果存入长期记忆，避免重复踩坑。

## 模块四：Multi-Agent 多智能体（中高阶考点，40K 加分项，追问 4~6 轮）
### 完整夺命追问链
1. 问：有没有落地过多智能体（Multi-Agent）？介绍架构和场景。
   - 追问1：主流几种协作模式：主从(Manager-Worker)、流水线、辩论博弈、对等协作，各自适用场景？
   - 追问2：你为什么选择当前架构？有没有对比过其他模式的优劣？
   - 追问3：多 Agent 之间怎么通信？消息格式、协议怎么设计？
   - 追问4：某个子 Agent 执行失败、超时、卡死，整个系统怎么容错？
   - 追问5：多 Agent 调用成本很高（多轮 LLM 调用），你做过哪些成本优化？
   - 追问6：如果业务扩张，需要新增几十类角色 Agent，架构怎么保证可扩展？

### 考察目的
考察**复杂系统设计、容错、扩展性、成本控制**，是迈向架构方向的关键能力。

### 高分核心思路
1. 选型：标准化流程选流水线、复杂综合业务选主从架构、高风险决策选辩论模式；
2. 通信：统一结构化消息（角色、任务、状态、结果、异常码），避免自由文本沟通；
3. 容错：任务超时重试、失败降级、任务重分配、熔断机制；
4. 成本：精简不必要的 Agent 角色、合并重复推理、压缩 Prompt、控制单轮 Token 数量。

## 模块五：工程化、稳定性、安全合规（线上生存能力，追问 4~5 轮）
### 完整夺命追问链
1. 问：你的 AI 应用是怎么部署上线的？整体服务架构？
   - 追问1：接口并发高的时候，怎么做限流、熔断、负载均衡？
   - 追问2：流式输出（SSE）怎么实现？线上遇到过哪些问题？
   - 追问3：全链路监控你搭建了哪些指标？怎么快速定位调用慢、报错、幻觉问题？
   - 追问4：AI 应用有哪些典型安全风险？提示注入怎么防御？
   - 追问5：涉及用户隐私/企业数据，你做了哪些数据安全和合规处理？

### 考察目的
AI 应用最终是线上服务，**只会算法不会工程的人薪资上限很低**，40K 要求具备完整线上运维&安全能力。

### 核心考点应答
1. 部署：FastAPI + Docker + 反向代理；高并发：接口限流、队列削峰、模型服务负载均衡；
2. 监控：LLM 调用耗时、Token 消耗、错误率、工具调用成功率、幻觉率、检索耗时；
3. 安全：输入检测隔离系统 Prompt、恶意指令拦截、工具最小权限、代码执行沙箱、数据脱敏、操作审计日志。

## 模块六：综合场景设计 + 方案选型（终面压轴题，自由发挥+深挖，决定最终薪资档位）
这类问题无标准答案，考察**综合架构思维、业务理解、权衡取舍能力**。

### 场景1：业务设计题（高频）
> 题目：现在要给一家电商商家做一站式 AI 运营助手，需求：自动数据分析、竞品分析、选品、文案生成、投放建议。请你从 0 设计整套方案。
> 连环追问：
> 1. 整体用单 Agent 还是多 Agent？为什么？
> 2. 哪些环节用 RAG？哪些环节需要工具调用？
> 3. 如何保证输出的运营建议准确、不编造数据？
> 4. 商家有上千条历史运营文档，怎么接入系统？
> 5. 预估 QPS 较高，架构上怎么优化性能与成本？

### 场景2：方案对比选型题
> 题目：传统 RAG、Graph RAG、自适应 RAG，分别在什么业务下优先选择？如果公司预算有限，怎么取舍？
> 追问：Graph RAG 开发成本更高，什么情况下必须硬上？

### 场景3：问题兜底题（压力测试）
> 题目：线上突然大批量出现幻觉、用户投诉暴增，你作为负责人，排查思路和应急方案是什么？
> 追问：如果是知识库更新不及时导致的，长期怎么根治？

### 场景4：行业趋势题（看视野）
> 题目：你认为当前 AI Agent 落地最大的瓶颈是什么？未来 1~2 年工程侧重点要解决什么问题？
> 追问：如果让你规划团队 AI 技术路线，优先级怎么排？

# 三、总结：40K 候选人 vs 初级候选人 核心区别（面试官评判标准）
| 能力维度 | 月薪 20~30K（初级） | 月薪 35~40K（中高级） |
| ---- | ---- | ---- |
| 框架使用 | 依赖框架，只会调用现成组件 | 理解框架原理，可自定义组件、甚至自研简易引擎 |
| 问题解决 | 遇到问题搜方案、堆功能 | 定位根因，设计体系化解决方案 |
