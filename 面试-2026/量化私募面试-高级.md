# 第一章：逻辑推理

## 1.1 命题推理

### 概念解释

**命题**是一个可以判断真假的陈述句。命题推理考察的是你根据已知命题推导出新命题的能力。

常见命题连接词：

| 连接词 | 符号 | 含义 | 例句 |
|--------|------|------|------|
| 且（合取） | ∧ / AND | 两者同时为真才为真 | "今天下雨且天冷" |
| 或（析取） | ∨ / OR | 至少一个为真即为真 | "买涨或买跌" |
| 非（否定） | ¬ / NOT | 取反 | "不是多头" |
| 如果…则…（蕴含） | → | 前件真则后件必真 | "如果突破均线，则买入" |
| 当且仅当（等价） | ↔ | 两者同真同假 | "盈利当且仅当净值>1" |

### 核心公式：真值表

**蕴含关系 A → B 的真值表**（最常考，也最容易错）：

| A | B | A → B |
|---|----|-------|
| T | T | T |
| T | F | **F** ← 唯一为假的情况 |
| F | T | T |
| F | F | T |

```python
# 非A或B=不能为真A假B
not A or B == not (A and not B) === not A or not (not B)
```

> 🔑 **关键记忆**：A → B 只有在「A 真且 B 假」时才为假。前件为假时，蕴含式恒真（虚真）。

**逆否命题（必考 ⭐️⭐️⭐️⭐️⭐️）**：
- 原命题：A → B
- 逆否命题：¬B → ¬A（与原命题**等价**）
- 逆命题：B → A（不等价！）
- 否命题：¬A → ¬B（不等价！）

### 详细讲解

**摩根定律（De Morgan's Laws）**——量化面试极高频率：

$$\neg (A \land B) \equiv \neg A \lor \neg B$$
$$\neg (A \lor B) \equiv \neg A \land \neg B$$

**翻译成人话**：
- "不是两者都成立" = "至少有一个不成立"
- "不是至少有一个成立" = "两个都不成立"

**量化场景应用**：
> "不是同时满足 ROE>15% 且 负债率<50% 才买入"
> 等价于："ROE≤15% 或 负债率≥50% 就不买入"
>
> ⚠️ **【批注——此例有逻辑错误】**：上述"等价"在严格命题逻辑下**不成立**。设 A=ROE>15%, B=负债率<50%, C=买入：
> - 原句（否定必要条件）：¬(C → (A∧B)) ≡ **C ∧ (¬A ∨ ¬B)** → 含义是"条件不满足也可以买"，即条件并非必须
> - 后句（不满足就不买）：(¬A ∨ ¬B) → ¬C ≡ **C → (A∧B)** → 含义是"要买必须满足条件"
>
> 两者互为正反，并不等价。摩根定律只作用于条件部分：¬(A∧B) ≡ ¬A∨¬B（纯逻辑正确），但"才/就不"引入的动作逻辑不能直接套摩根定律。正确示例：
> - 纯摩根定律："不满足(ROE>15% 且 负债率<50%)" ≡ "ROE≤15% 或 负债率≥50%" ✅
> - 带动作等价："不是满足条件才买" ≡ "不满足条件也买"，而非"不满足条件就不买"

### 🐍 Python 实战：真值表与命题演算

```python
"""
命题逻辑工具：真值表生成 & 摩根定律验证
面试中经常需要你手写验证逻辑——建议熟练掌握以下写法
"""

from itertools import product
from typing import Callable


def truth_table(expr: Callable[[bool, bool], bool],
                names: tuple[str, str] = ("A", "B")) -> None:
    """
    打印二元逻辑表达式的真值表

    Args:
        expr: 接受两个 bool 参数、返回 bool 的函数
        names: 变量名，用于表头显示

    面试技巧：面试官可能让你手写蕴含关系的真值表，
    用代码思维验证可以避免笔误
    """
    headers = f"{names[0]:^6} | {names[1]:^6} | Result"
    print(f"\n{'='*30}")
    print(f"  真值表：{expr.__doc__ or expr.__name__}")
    print(f"{'='*30}")
    print(headers)
    print("-" * 30)
    for a, b in product([True, False], repeat=2):
        print(f"{str(a):^6} | {str(b):^6} | {expr(a, b)}")
    print(f"{'='*30}\n")


# --- 定义逻辑连接词 ---

def implies(a: bool, b: bool) -> bool:
    """蕴含 A → B: 只有 A真且B假 时为假"""
    return not a or b  # 等价写法：¬A ∨ B


def conjunction(a: bool, b: bool) -> bool:
    """合取 A ∧ B: 两者同时为真"""
    return a and b


def disjunction(a: bool, b: bool) -> bool:
    """析取 A ∨ B: 至少一个为真"""
    return a or b


def equivalence(a: bool, b: bool) -> bool:
    """等价 A ↔ B: 同真同假"""
    return a == b


# --- 验证摩根定律 ---

def verify_de_morgan() -> None:
    """
    用穷举法验证摩根定律：
      ¬(A ∧ B) ≡ ¬A ∨ ¬B
      ¬(A ∨ B) ≡ ¬A ∧ ¬B

    核心面试点：摩根定律在量化中的高频应用——
    否定"且"得到"或否定"，否定"或"得到"且否定"
    """
    print("=" * 60)
    print("  摩根定律穷举验证（De Morgan's Laws Verification）")
    print("=" * 60)
    for a, b in product([True, False], repeat=2):
        # 定律1: ¬(A ∧ B) ≡ ¬A ∨ ¬B
        left1 = not (a and b)
        right1 = (not a) or (not b)

        # 定律2: ¬(A ∨ B) ≡ ¬A ∧ ¬B
        left2 = not (a or b)
        right2 = (not a) and (not b)

        assert left1 == right1, f"摩根定律1验证失败: A={a}, B={b}"
        assert left2 == right2, f"摩根定律2验证失败: A={a}, B={b}"

        print(f"  A={a}, B={b}  |  ¬(A∧B)={left1} ≡ ¬A∨¬B={right1}"
              f"  |  ¬(A∨B)={left2} ≡ ¬A∧¬B={right2}")
    print("  ✅ 摩根定律全验证通过！")
    print("=" * 60 + "\n")


# --- 逆否命题等价性验证 ---

def verify_contrapositive() -> None:
    """
    验证：原命题 (A → B) 与 逆否命题 (¬B → ¬A) 等价
    同时验证：逆命题 (B → A) 和否命题 (¬A → ¬B) 不等价
    """
    print("=" * 60)
    print("  逆否命题等价性验证")
    print("=" * 60)
    print(f"{'A':^6} {'B':^6} {'A→B':^6} {'¬B→¬A':^6} {'等价?':^8} {'B→A':^6} {'¬A→¬B':^6}")
    print("-" * 60)
    for a, b in product([True, False], repeat=2):
        original = implies(a, b)
        contrapositive = implies(not b, not a)  # 逆否
        converse = implies(b, a)                 # 逆命题
        inverse = implies(not a, not b)          # 否命题

        assert original == contrapositive, "逆否命题必须等价！"
        print(f"{str(a):^6} {str(b):^6} {str(original):^6} {str(contrapositive):^6}"
              f" {'✅' if original == contrapositive else '❌':^8}"
              f" {str(converse):^6} {str(inverse):^6}")
    print("  结论：原命题 ≡ 逆否命题；逆命题/否命题不保证等价")
    print("=" * 60 + "\n")


# 运行验证
if __name__ == "__main__":
    truth_table(implies, ("A", "B"))
    verify_de_morgan()
    verify_contrapositive()
```

**题2**：某基金经理说："如果市盈率低于10，且净利润增长超过20%，我就买入。"现在他没有买入。以下哪项一定为真？
- A. 市盈率不低于10
- B. 净利润增长不超过20%
- C. 市盈率不低于10 或 净利润增长不超过20%
- D. 市盈率不低于10 且 净利润增长不超过20%

<details>
<summary>点击查看答案</summary>

**答案：C**

设 P = 市盈率低于10，Q = 净利润增长超过20%，R = 买入。

已知：(P ∧ Q) → R，且 ¬R 为真。

由逆否命题：¬R → ¬(P ∧ Q)

由摩根定律：¬(P ∧ Q) ≡ ¬P ∨ ¬Q

即："市盈率不低于10 或 净利润增长不超过20%"

注意 C 是"或"不是"且"，D 犯了把否定"且"当成"且否定"的错误。
</details>

## 1.2 条件判断

### 概念解释

条件判断题给出若干条件和约束，要求你判断某个结论是否必然成立，或找出满足所有条件的方案。这类题核心是**约束满足问题（CSP）**。

### 核心方法

**方法一：穷举法（适合小规模）**

画出所有可能情况，逐一排除。

**方法二：矛盾法**

假设结论不成立，推导矛盾。

**方法三：列表/矩阵法**

适用于多条件交叉判断。

### 详细讲解

**经典题型——谁说谎问题**：

> 甲说：乙在说谎。
> 乙说：丙在说谎。
> 丙说：甲和乙都在说谎。
> 问：谁说真话？

**解题步骤**：

1. 假设甲说真话 → 乙说谎 → 丙说真话（因为乙说"丙说谎"是假的）→ 丙说"甲和乙都说谎"，但甲说真话，矛盾！
2. 假设甲说谎 → 乙说真话 → 丙说谎 → 丙说"甲和乙都说谎"为假，但甲确实说谎、乙说真话，所以"都"字使该命题为假 ✓

**答案**：乙说真话，甲和丙说谎。

### 🐍 Python 实战：穷举法解谁说谎问题

```python
"""
谁说谎问题：用 Python 穷举法暴力搜索所有可能性

面试常见的"谁说谎"问题，推广到 N 个人时手动推理困难，
但代码可以在 O(2^N) 内穷举出所有解
"""

from itertools import product
from typing import List, Tuple


def solve_liar_puzzle() -> List[Tuple[bool, bool, bool]]:
    """
    甲说：乙在说谎       -> 甲真 ⟺ 乙假
    乙说：丙在说谎       -> 乙真 ⟺ 丙假
    丙说：甲和乙都在说谎 -> 丙真 ⟺ (甲假 ∧ 乙假)

    Returns:
        所有满足条件的三元组 (甲说真话, 乙说真话, 丙说真话)
    """
    solutions = []

    for a, b, c in product([True, False], repeat=3):
        # 甲的陈述："乙在说谎"
        a_statement = (not b)
        # 乙的陈述："丙在说谎"
        b_statement = (not c)
        # 丙的陈述："甲和乙都在说谎"
        c_statement = (not a and not b)

        # 关键：说话者说真话 ⟺ 其陈述为真
        if (a == a_statement) and (b == b_statement) and (c == c_statement):
            solutions.append((a, b, c))

    return solutions


def pretty_print_solutions(solutions: List[Tuple[bool, bool, bool]]) -> None:
    """格式化输出结果"""
    if not solutions:
        print("❌ 无解！检查题设是否有误。")
        return

    print(f"\n找到 {len(solutions)} 组解：\n")
    for i, (a, b, c) in enumerate(solutions, 1):
        truth_tellers = []
        liars = []
        for name, tells_truth in [("甲", a), ("乙", b), ("丙", c)]:
            if tells_truth:
                truth_tellers.append(name)
            else:
                liars.append(name)
        print(f"  解{i}: 说真话 → {', '.join(truth_tellers) or '无'}"
              f"  |  说谎 → {', '.join(liars) or '无'}")


# 运行
solutions = solve_liar_puzzle()
pretty_print_solutions(solutions)
```

### 🔥 资深面试策略：逻辑题的潜在陷阱

面试官在逻辑题中通常会留一个"坑"，看你是否踩进去：

1. **"有些" ≠ "所有"**：看到"有些 S 是 P"时，立即警觉——这不意味着"有些 S 不是 P"
2. **蕴含的前件为假**：当条件句的前提不成立时，条件句本身是真。很多人理解不了"A 假 → A→B 真"
3. **必要条件和充分条件的混淆**："只有 A 才 B" = B→A（不是 A→B）
4. **"至少"和"恰好"的区别**：5人中至少 2 人说谎 ≠ 恰好 2 人说谎

**资深回答模板**：当面试官问"你确定吗？"，不要立刻改答案。先冷静说出你的推理链条，然后检查每一步的前提。如果发现漏洞，大方承认并修正——这比死守错误答案要好得多。

**题3**：有 A、B、C 三只股票，已知：
1. 如果 A 涨，则 B 涨
2. 如果 B 涨，则 C 跌
3. 如果 C 跌，则 A 涨

问：以下哪种情况不可能发生？
- A. A涨、B涨、C跌
- B. A跌、B跌、C涨
- C. A跌、B跌、C跌
- D. A涨、B跌、C跌

<details>
<summary>点击查看答案</summary>

**答案：D**

逐项验证：
- A：A涨→B涨（满足），B涨→C跌（满足），C跌→A涨（满足）✓
- B：A跌，条件1前件假，恒真。B跌，条件2前件假，恒真。C涨，条件3前件假，恒真。✓
- C：同理，全部前件为假，恒真 ✓
- D：A涨→B必须涨，但B跌，违反条件1 ✗

所以 D 不可能发生。
</details>

**🐍 拓展**：上面这题也可以用 Python 穷举验证：

```python
def check_stock_conditions(a_up: bool, b_up: bool, c_down: bool) -> bool:
    """检查股票涨跌是否同时满足三个条件"""
    cond1 = not a_up or b_up           # A涨 → B涨
    cond2 = not b_up or c_down         # B涨 → C跌
    cond3 = not c_down or a_up         # C跌 → A涨
    return cond1 and cond2 and cond3

# 穷举所有 8 种情况
for a, b, c in product([True, False], repeat=3):
    valid = check_stock_conditions(a, b, c)
    status = "✅ 可能" if valid else "❌ 不可能"
    print(f"A涨={a}, B涨={b}, C跌={c} → {status}")
```

**题4**：5个人分100枚金币，每人投一票（包括提议者），投票决定是否通过分配方案。如果通过则按方案分，不通过则提议者出局，剩下的人继续。你是第1个提议者，假设所有人绝对理性且贪婪，你最多能给自己分多少？

<details>
<summary>点击查看答案</summary>

**答案：98枚**

经典的海盗分金问题变体。倒推法：

- 只剩2人(4,5)：4给自己100、给5为0。投票1:1（4赞成、5反对），平票不通过？要看规则。若平票也算不通过，则4出局，5拿100。所以4必须给5至少1枚（如果5投反对则4出局5全拿，5应该反对），但实际上如果规则是平票通过，则4给0即可。
- 按最常见规则（平票通过）：只剩2人时4给自己100、5给0，4自己一票即通过。
- 剩3人(3,4,5)：3需要再争取1票。给5只需1枚（因为5在下一轮得0），给自己99、给4为0、给5为1。

```python
from itertools import permutations

def solve_ranking():
    """穷举四人排名，筛选满足所有条件的排列"""
    people = ["甲", "乙", "丙", "丁"]
    solutions = []

    # 生成所有排列：p[0]=第1名(最高), p[1]=第2名, ..., p[3]=第4名(最低)
    for p in permutations(people):
        # 条件1: 甲不是最高(idx!=0) 也不是最低(idx!=3)
        if p.index("甲") in (0, 3):
            continue
        # 条件2: 乙比丙高（乙的 idx < 丙的 idx）
        if p.index("乙") >= p.index("丙"):
            continue
        # 条件3: 丁比甲低（丁的 idx > 甲的 idx）
        if p.index("丁") <= p.index("甲"):
            continue
        # 条件4: 丙不是第三名(idx!=2)
        if p.index("丙") == 2:
            continue
        solutions.append(p)

    return [(f"{'>'.join(p)}", p) for p in solutions]


for rank_str, _ in solve_ranking():
    print(f"  {rank_str}")
# 输出:
#   乙>甲>丁>丙
#   乙>丙>甲>丁
```

## 1.4 概率计算（逻辑题）

### 概念解释

将概率思维应用于逻辑推理中，常见题型包括条件概率、贝叶斯更新、随机事件逻辑。

### 核心公式

**条件概率**（贝叶斯公式的基础）：

$$P(A|B) = \frac{P(A \cap B)}{P(B)}$$

**贝叶斯公式**（极其重要，量化面试必考）：

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)} = \frac{P(B|A) \cdot P(A)}{P(B|A)P(A) + P(B|\neg A)P(\neg A)}$$

**全概率公式**：

$$P(B) = \sum_{i} P(B|A_i) \cdot P(A_i)$$

### 详细讲解

**贝叶斯公式在量化中的应用**：

> 你有一个交易策略，历史回测表明：
> - 在牛市环境中，策略盈利的概率是 80%
> - 在熊市环境中，策略盈利的概率是 30%
> - 当前市场是牛市的概率（先验）是 60%
>
> 今天策略发出了盈利信号，问当前市场是牛市的概率是多少？

**解答**：

$$P(\text{牛市}|\text{盈利}) = \frac{P(\text{盈利}|\text{牛市}) \cdot P(\text{牛市})}{P(\text{盈利})}$$

$$P(\text{盈利}) = 0.8 \times 0.6 + 0.3 \times 0.4 = 0.48 + 0.12 = 0.60$$

$$P(\text{牛市}|\text{盈利}) = \frac{0.8 \times 0.6}{0.60} = \frac{0.48}{0.60} = 0.80$$

**解读**：策略盈利后，牛市概率从 60% 更新为 80%，这就是贝叶斯更新。

### 🐍 Python 实战：贝叶斯更新

```python
"""
贝叶斯公式 Python 实现
面试高频考点：给定先验概率和条件概率，计算后验概率
"""


def bayes_update(
    prior: float,           # P(A): 先验概率
    likelihood_given: float,  # P(B|A): A发生时B发生的概率
    likelihood_given_not: float,  # P(B|¬A): A不发生时B发生的概率
) -> float:
    """
    贝叶斯更新：计算 P(A|B)

    Returns:
        后验概率 P(A|B)

    Example:
        >>> # 牛市先验 60%，策略在牛市盈利 80%，熊市盈利 30%
        >>> bayes_update(prior=0.6, likelihood_given=0.8, likelihood_given_not=0.3)
        0.8
    """
    # P(B) = P(B|A)·P(A) + P(B|¬A)·P(¬A)  — 全概率公式
    p_b = likelihood_given * prior + likelihood_given_not * (1 - prior)

    # P(A|B) = P(B|A)·P(A) / P(B)  — 贝叶斯公式
    posterior = (likelihood_given * prior) / p_b
    return posterior


# --- 经典面试题：罕见病检测 ---
def disease_test_example() -> None:
    """
    罕见病检测——基础率谬误（Base Rate Fallacy）

    面试必考题：发病率 0.1%，检测准确率 99%，
    阳性结果下真正患病的概率是多少？
    """
    # 已知条件
    p_disease = 0.001         # 先验：发病率 0.1%
    p_positive_given_disease = 0.99   # 灵敏度：有病检出阳性
    p_positive_given_healthy = 0.01   # 假阳性率：没病检出阳性

    # 贝叶斯更新
    p_disease_given_positive = bayes_update(
        prior=p_disease,
        likelihood_given=p_positive_given_disease,
        likelihood_given_not=p_positive_given_healthy,
    )

    print("=" * 50)
    print("  罕见病检测——基础率谬误（Base Rate Fallacy）")
    print("=" * 50)
    print(f"  先验概率 P(患病)           = {p_disease:.3f} ({p_disease*100:.1f}%)")
    print(f"  灵敏度 P(阳性|患病)         = {p_positive_given_disease:.2f}")
    print(f"  假阳性率 P(阳性|健康)       = {p_positive_given_healthy:.2f}")
    print(f"  ─────────────────────────────")
    print(f"  后验概率 P(患病|阳性)       = {p_disease_given_positive:.4f}"
          f" ({p_disease_given_positive*100:.2f}%)")
    print()
    print("  ⚠️  为什么这么低？因为疾病本身极罕见（0.1%），")
```

**题6**（经典面试题）：一种罕见病在人群中的发病率是 0.1%。检测方法的准确率是 99%（有病测出阳性概率 99%，没病测出阴性概率 99%）。如果一个人检测结果为阳性，他真正患病的概率是多少？

<details>
<summary>点击查看答案</summary>

**答案：约 9.02%**

这是经典的**基础率谬误**问题。

- P(患病) = 0.001
- P(阳性|患病) = 0.99
- P(阳性|健康) = 0.01

$$P(\text{患病}|\text{阳性}) = \frac{0.99 \times 0.001}{0.99 \times 0.001 + 0.01 \times 0.999}$$

$$= \frac{0.00099}{0.00099 + 0.00999} = \frac{0.00099}{0.01098} \approx 9.02\%$$

**为什么这么低？** 因为疾病本身极其罕见，即使检测准确率很高，大多数阳性结果仍然是假阳性。这个思维在量化交易中非常重要——**信号的高准确率不等于策略的高胜率，还要看基础概率**。

### 🔥 资深面试策略：贝叶斯题目的深度追问

面试官问完基础贝叶斯题后，资深岗常追加以下问题：

**追问1**：如果连续两次检测都是阳性，患病概率是多少？
→ 第一次后验 (9.02%) 成为第二次的**先验**。P(患病|两次阳性) = (0.99²×0.001) / (0.99²×0.001 + 0.01²×0.999) ≈ 90.7%

**追问2**：贝叶斯更新和机器学习中的"在线学习"有什么联系？
→ 这是**sequential Bayesian updating**——每次新数据到来时，后验成为新先验。等同于在线梯度下降的思想。

**追问3**：在量化策略中，你如何用贝叶斯框架来更新策略参数？
→ 核心考点：把历史回测当作先验，实盘表现作为新证据，持续更新对策略有效性的判断。如果实盘 Sharpe 持续低于预期，后验会逐步倾向于"策略失效"。

## 1.5 削弱加强

### 概念解释

给出一段论证（前提→结论），要求选择最能**削弱**（使结论不成立或可能性降低）或**加强**（使结论更可信）的选项。核心是找到论证的**逻辑漏洞**或**隐含假设**。

### 核心方法

**削弱的三条路径**：
1. **否定前提**：攻击论据本身
2. **切断因果**：前提推不出结论（他因导致、因果倒置、有因无果等）
3. **否定结论**：直接证明结论为假

**加强的三条路径**：
1. **加固前提**：提供额外证据支持论据
2. **加固推理**：排除他因、证明因果方向、提供类比
3. **直接支持结论**：提供独立证据

### 配套考题

**题7**：某量化研究员发现，每当社交媒体上关于某只股票的讨论量暴增时，该股票在接下来一周内上涨的概率是 65%。他得出结论：社交媒体热度**导致**了股价上涨。

以下哪项最能**削弱**这个结论？
- A. 社交媒体热度增加时，也可能是股价已经上涨引发的讨论
- B. 该研究员只分析了科技板块的股票
- C. 有些股票讨论量暴增后并没有上涨
- D. 社交媒体上的讨论大部分是非专业人士发表的

<details>
<summary>点击查看答案</summary>

**答案：A**

A 指出**因果倒置**——不是社交媒体导致上涨，而是上涨导致了社交媒体讨论。这是削弱因果推理最有力的方式。

B 是样本偏差，力度中等。
C 说"有些"，只是削弱了必然性，但原结论用的是"概率 65%"，并不排斥个别反例。
D 与推理链条无关。
</details>

## 1.6 博弈策略

### 概念解释

博弈论研究多个理性决策者之间的策略互动。量化面试中常考**纳什均衡**、**囚徒困境**、**混合策略**、**零和博弈**。

### 核心概念与公式

**纳什均衡**：每个参与者在给定他人策略下，都没有动机单方面改变自己的策略。

**混合策略纳什均衡**：

在 2×2 博弈中，设参与人1选A的概率为 p，选B的概率为 1-p。参与人2选A和选B的期望收益相等时，p 即为均衡概率。

$$\mathbb{E}[\text{参与人2选A}] = \mathbb{E}[\text{参与人2选B}]$$

**期望值公式**：

$$\mathbb{E}[X] = \sum_i p_i \cdot x_i$$

### 详细讲解

**经典例题——猜数字博弈**：

> N 个人各选一个 0~100 之间的整数。目标：猜中**所有人所选数字的平均值的 2/3**。最接近者获胜。你会选什么数字？

**推理层次**：
- **第0层**（完全不思考）：随机选，平均值约50，2/3 ≈ 33
- **第1层**（认为别人是第0层）：选 33 的 2/3 ≈ 22
- **第2层**（认为别人是第1层）：选 22 的 2/3 ≈ 15
- **第N层**（无限递归）：选 0 或 1

**真正的最优策略**取决于你对其他参与者推理层次的判断。这反映了**预测市场行为需要判断博弈参与者有多"聪明"**，与量化交易中预测市场非常相似。

### 🐍 Python 实战：猜数字博弈模拟

```python
"""
猜数字博弈（Beauty Contest / Keynesian Beauty Contest）
N 个人选 0~100，目标是所有人平均值的 2/3

这是理解"市场预期"的经典博弈论模型
"""

import random
from typing import List


def k_level_choice(k: int) -> float:
    """
    K 层推理者的选择

    Args:
        k: 推理层数 (k=0: 随机, k=1: 以为别人是 k=0, ...)

    Returns:
        该层推理者会选择的最优数字

    面试要点：这展示了"iterated elimination of dominated strategies"
    ——每多一层推理，数字更接近 0
    """
    if k <= 0:
        # 第 0 层：随机选择（或假设均匀分布的平均值 50）
        return 50.0

    # 第 k 层：认为所有人是第 k-1 层，选其 2/3
    return k_level_choice(k - 1) * 2 / 3


def simulate_beauty_contest(n_players: int = 100,
                            k_distribution: List[float] = None) -> dict:
    """
    模拟猜数字博弈

    Args:
        n_players: 参与者人数
        k_distribution: 各推理层级的玩家比例
                        默认 [0.2, 0.3, 0.3, 0.15, 0.05]（0~4层）

    Returns:
        包含平均选择、2/3目标、赢家信息的字典
    """
    if k_distribution is None:
        k_distribution = [0.2, 0.3, 0.3, 0.15, 0.05]

    choices = []
    for k, proportion in enumerate(k_distribution):
        count = int(n_players * proportion)
        base = k_level_choice(k)
        # 加入少量噪声模拟真实行为
        choices.extend([base + random.uniform(-2, 2) for _ in range(count)])

    # 确保恰好 n_players 人
    choices = choices[:n_players]
    while len(choices) < n_players:
        choices.append(50 + random.uniform(-5, 5))

    avg = sum(choices) / len(choices)
    target = avg * 2 / 3

    # 找出最接近目标的赢家
    winner_idx = min(range(len(choices)),
                     key=lambda i: abs(choices[i] - target))

    return {
        "choices": choices,
        "average": avg,
        "target": target,
        "winner_choice": choices[winner_idx],
        "winner_idx": winner_idx,
    }


# 演示 K 层推理的收敛
print("K 层推理的猜数字博弈选择：")
print(f"{'层级':<6} {'选择':<12} {'推理'}")
print("-" * 40)
for k in range(7):
    print(f"  K={k:<3}  {k_level_choice(k):>8.2f}     "
          f"{'→ 无限递归收敛到 0' if k > 4 else ''}")

# 运行一次模拟
result = simulate_beauty_contest(n_players=50)
print(f"\n模拟结果：平均={result['average']:.2f}, "
      f"2/3目标={result['target']:.2f}, "
      f"赢家选择={result['winner_choice']:.2f}")
```

### 配套考题

**题8**（量化面试高频题）：A 和 B 各写一个整数 1~10。如果两人数字相同，各得 10 元；如果相差 1，数字小的人得 15 元、大的人得 5 元；如果相差超过 1，各得 0 元。求纯策略纳什均衡。

<details>
<summary>点击查看答案</summary>

**答案：(1,1) 是唯一的纯策略纳什均衡。**

构造收益矩阵（部分）：

| A\B | 1 | 2 | 3 | ... |
|------|----|----|----|-----|
| 1 | 10,10 | 15,5 | 0,0 | ... |
| 2 | 5,15 | 10,10 | 15,5 | ... |
| 3 | 0,0 | 5,15 | 10,10 | ... |

规则回顾：相差 1 时，**数字小的**得 15、**大的**得 5。所以矩阵中 "15,5" 表示行玩家选了更小的数（得15），列玩家选了更大的数（得5）。

**检查 (1,1)**：
- B 面对 A=1：B=1 得 10，B=2 得 5（B 是大的那个）→ **不偏离**
- A 面对 B=1：A=1 得 10，A=2 得 5（A 是大的那个）→ **不偏离**
→ (1,1) 是纳什均衡 ✓

**为什么其他策略不是均衡**：对任意 k ≥ 2，当对手选 k 时，你选 k-1 可得 15 元（你是小的那个），优于选 k 只得 10 元——所以任何 k ≥ 2 的策略都会被 "选小一号" 偏离掉。只有 (1,1) 因为不存在比 1 更小的数，才无法被这样偏离。

**(1,1) 是唯一的纯策略纳什均衡。** 这道题不需要混合策略——在有限策略博弈中，是存在纯策略均衡的经典例子。
</details>

## 1.7 数字规律

### 概念解释

### 常见规律类型

| 类型 | 特征 | 例子 |
|------|------|------|
| 等差 | 相邻差相等 | 2, 5, 8, 11, 14... |

| 差后等差 | 差值构成等差 | 1, 2, 4, 7, 11, 16... |
| 质数 | 质数序列 | 2, 3, 5, 7, 11, 13... |
| 交错 | 奇偶位不同规则 | 2, 5, 4, 10, 6, 15... |
| 自描述 | 涉及数字本身性质 | 1, 11, 21, 1211, 111221... |

### 🐍 Python 实战：常见数列生成器

```python
"""
数字规律——常见数列的 Python 实现
面试中可能让你手写某个数列的生成函数
"""

from typing import List, Iterator


def arithmetic_sequence(a1: float, d: float, n: int) -> List[float]:
    """等差数列：a₁, a₁+d, a₁+2d, ..."""
    return [a1 + i * d for i in range(n)]


def geometric_sequence(a1: float, q: float, n: int) -> List[float]:
    """等比数列：a₁, a₁·q, a₁·q², ..."""
    return [a1 * (q ** i) for i in range(n)]


def squares(n: int) -> List[int]:
    """平方数列：1², 2², 3², ..."""
    return [i * i for i in range(1, n + 1)]


def fibonacci(n: int) -> List[int]:
    """斐波那契数列：1, 1, 2, 3, 5, 8, ..."""
    if n <= 0:
        return []
    seq = [1, 1]
    for _ in range(2, n):
        seq.append(seq[-1] + seq[-2])
    return seq[:n]


def triangular_numbers(n: int) -> List[int]:
    """三角形数：0, 1, 3, 6, 10, 15, ..., T_k = k(k-1)/2"""
    return [k * (k - 1) // 2 for k in range(n)]


def factorial_sequence(n: int) -> List[int]:
    """阶乘序列：1!, 2!, 3!, ..."""
    result = []
    f = 1
    for i in range(1, n + 1):
        f *= i
        result.append(f)
    return result


def look_and_say(n_terms: int) -> List[str]:
    """
    外观数列（Look-and-Say Sequence）—— LeetCode #38

    面试高频题！每个数是对上一个数的"读法"：
    1 → "一个1" → 11
    11 → "两个1" → 21
    21 → "一个2, 一个1" → 1211
    """
    if n_terms <= 0:
        return []

    seq = ["1"]
    for _ in range(1, n_terms):
        prev = seq[-1]
        result = []
        i = 0
        while i < len(prev):
            count = 1
            while i + count < len(prev) and prev[i + count] == prev[i]:
                count += 1
            result.append(f"{count}{prev[i]}")
            i += count
        seq.append("".join(result))
    return seq


# 演示
print("等差数列 (a1=2, d=3):", arithmetic_sequence(2, 3, 6))
print("等比数列 (a1=3, q=2):", geometric_sequence(3, 2, 6))
print("平方数列:", squares(6))
print("斐波那契:", fibonacci(10))
print("三角形数:", triangular_numbers(7))
print("阶乘:", factorial_sequence(6))
print("外观数列:", look_and_say(6))
```

## 1.8 逻辑矛盾

### 概念解释

从一组陈述中找出互相矛盾的命题，利用矛盾关系锁定真假。核心工具是**矛盾律**（A 和 ¬A 不能同时为真）和**排中律**（A 和 ¬A 必有一个为真）。

### 核心方法

**矛盾对**识别——以下关系的两句话不能同时为真也不能同时为假：
- "所有 S 都是 P" vs "有些 S 不是 P"
- "所有 S 都不是 P" vs "有些 S 是 P"
- "A 且 B" vs "非 A 或非 B"
- "A 或 B" vs "非 A 且非 B"

**突破口技巧**：
1. 找到互相矛盾的两句话 → 必有一真一假
2. 利用"只有一句真话"等约束 → 缩小范围
3. 假设验证 → 逐一排除

```python
def solve_truth_teller_puzzle(
    statements: List[Callable[[Tuple[bool, ...]], bool]],
    n_true: int = 1,           # 恰好几人说真话
    names: List[str] = None    # 说话者名称
) -> List[Tuple[bool, ...]]:

    通用"谁说真话"求解器

    Args:
        statements: 各人的陈述函数，入参为所有人的真假状态元组
        n_true: 说真话的人数约束
        names: 说话者名称列表

    Returns:
        所有满足条件的真假状态元组
    """
    n = len(statements)
    if names is None:
        names = [f"Person_{i+1}" for i in range(n)]

    solutions = []
    for state in product([True, False], repeat=n):
        # 检查"恰好 n_true 人说真话"
        if sum(state) != n_true:
            continue

        for i, stmt in enumerate(statements):

            # 如果此人说谎(state[i]=False)，其陈述必须为假
            if state[i] != stmt(state):
                valid = False
                break

        if valid:
            solutions.append(state)

    return solutions
```

## 2.1 加权平均

### 概念解释

不同数据点有不同权重时，使用加权平均而不是简单平均。量化面试中的重要应用：组合收益率、仓位加权、时间加权收益率等。

### 核心公式

$$\text{加权平均} = \frac{\sum_{i=1}^{n} w_i \cdot x_i}{\sum_{i=1}^{n} w_i}$$

当权重和为 1 时（归一化权重）：

$$\bar{x}_w = \sum_{i=1}^{n} w_i \cdot x_i, \quad \sum w_i = 1$$

**投资组合收益率**（权重为各资产占比）：

$$R_p = \sum_{i=1}^{n} w_i \cdot R_i$$

**时间加权收益率（TWR）**：

$$TWR = (1+R_1)(1+R_2)...(1+R_n) - 1$$

### 详细讲解

**不同"平均"的适用场景**：

| 平均类型 | 公式 | 适用场景 |
|----------|------|----------|
| 算术平均 | (a+b)/2 | 等权情况 |
| 加权平均 | Σ(wx)/Σw | 不等权 |
| 几何平均 | √(ab) | 比率、增长率 |
| 调和平均 | 2ab/(a+b) | 速率、PE |

### 🐍 Python 实战：各类平均的计算

```python
"""
不同"平均"在 Python 中的实现
量化面试中经常需要区分算术/加权/几何/调和平均
"""

import math
from typing import List, Sequence


def arithmetic_mean(values: Sequence[float]) -> float:
    """算术平均：等权求和除以个数"""
    return sum(values) / len(values)


def weighted_mean(values: Sequence[float],
                  weights: Sequence[float]) -> float:
    """加权平均：Σ(w_i · x_i) / Σw_i"""
    if len(values) != len(weights):
        raise ValueError("values 和 weights 长度必须一致")
    return sum(w * x for w, x in zip(weights, values)) / sum(weights)


def geometric_mean(values: Sequence[float]) -> float:
    """
    几何平均：n 个正数的乘积开 n 次方

    适用场景：复合收益率、同比增长率的平均
    易错点：不能用算术平均代替！
    """
    if any(v <= 0 for v in values):
        raise ValueError("几何平均要求所有值为正")
    # 用对数避免浮点溢出：exp(Σln(x)/n)
    log_sum = sum(math.log(v) for v in values)
    return math.exp(log_sum / len(values))


def harmonic_mean(values: Sequence[float]) -> float:
    """
    调和平均：n / Σ(1/x_i)

    适用场景：速度、市盈率、等额定投的平均成本
    经典题：往返速度问题 → 平均速度 = 调和平均
    """
    if any(v == 0 for v in values):
        raise ValueError("调和平均要求所有值非零")
    return len(values) / sum(1 / v for v in values)


def twr(period_returns: Sequence[float]) -> float:
    """
    时间加权收益率（Time-Weighted Return）

    TWR = ∏(1+R_i) - 1
    这是基金业绩评价的标准方法
    """
    product = 1.0
    for r in period_returns:
        product *= (1 + r)
    return product - 1


# --- 演示 ---

# 往返速度问题（经典易错题）
speed_to = 60     # 去程 60 km/h
speed_back = 40   # 回程 40 km/h

print("往返速度问题：")
print(f"  算术平均: {arithmetic_mean([speed_to, speed_back]):.1f} km/h  ← 错误答案")
print(f"  调和平均: {harmonic_mean([speed_to, speed_back]):.1f} km/h  ← 正确答案")
print()

# 投资组合
returns = [0.15, -0.05]   # A股 +15%, B股 -5%
weights = [0.6, 0.4]      # 仓位 60%, 40%
print(f"组合加权收益率: {weighted_mean(returns, weights):.2%}")
print(f"时间加权收益率: {twr([0.005, -0.003, 0.008, 0.012, -0.006]):.4%}")
```

## 2.2 比例计算

### 概念解释

量化面试中比例计算常与金融场景结合：收益率、杠杆率、夏普比率、胜率/盈亏比等。

总结：盈亏比才是炒股要把握好的技能。

### 核心公式

**百分比变化**：

$$\text{变化率} = \frac{\text{新值} - \text{旧值}}{\text{旧值}} \times 100\%$$

**连续复利**：

$$A = P \cdot e^{rt}$$

**杠杆收益率**：

$$R_{\text{杠杆}} = L \times R_{\text{资产}} - (L-1) \times r_{\text{融资}}$$

其中 L 为杠杆倍数，r 为融资成本。

### 🐍 Python 实战：交易指标计算

```python
"""
交易指标计算：盈亏比、期望收益、杠杆收益、连续复利
"""

import math


def percent_change(old: float, new: float) -> float:
    """百分比变化率"""
    return (new - old) / old


def expected_return(win_rate: float, avg_win: float,
                    avg_loss: float) -> float:
    """
    策略期望收益

    Args:
        win_rate: 胜率 (0~1)
        avg_win: 平均盈利 (正数，如 0.03 表示 3%)
        avg_loss: 平均亏损 (正数，如 0.01 表示 1%)

    Returns:
        每笔交易的期望收益率

    Example:
        >>> expected_return(0.4, 0.03, 0.01)
        0.006  # 0.6% 正期望
    """
    return win_rate * avg_win + (1 - win_rate) * (-avg_loss)


def profit_loss_ratio(avg_win: float, avg_loss: float) -> float:
    """盈亏比 = 平均盈利 / 平均亏损"""
    return avg_win / avg_loss


def leveraged_return(asset_return: float, leverage: float,
                     funding_rate: float) -> float:
    """
    杠杆收益率

    R_leveraged = L × R_asset - (L-1) × r_funding

    注意：杠杆放大了收益也放大了亏损
    如果 R_asset < r_funding，加杠杆会加速亏损
    """
    return leverage * asset_return - (leverage - 1) * funding_rate


def continuous_compounding(principal: float, rate: float,
                           years: float) -> float:
    """连续复利：A = P × e^(rt)"""
    return principal * math.exp(rate * years)


def rule_of_72(annual_return_percent: float) -> float:
    """
    72 法则：快速估算本金翻倍所需年数

    年化收益率 r% → 翻倍约需 72/r 年
    这是面试中必须掌握的速算技巧
    """
    return 72 / annual_return_percent


# 演示
print("策略分析：")
print(f"  盈亏比: {profit_loss_ratio(0.03, 0.01):.0f}:1")
print(f"  期望收益: {expected_return(0.4, 0.03, 0.01):.2%}")
print(f"  72法则 (8%年化): 约 {rule_of_72(8):.1f} 年翻倍")
print(f"  连续复利 (10万, 8%, 10年): "
      f"{continuous_compounding(100000, 0.08, 10):,.0f} 元")
```

## 2.3 排列组合

### 概念解释

量化面试的绝对重点。排列（有顺序）vs 组合（无顺序）必须区分清楚。

### 核心公式

**排列**（从 n 个中选 r 个，有顺序）：

$$P(n, r) = \frac{n!}{(n-r)!}$$

**组合**（从 n 个中选 r 个，无顺序）：

$$C(n, r) = \binom{n}{r} = \frac{n!}{r!(n-r)!}$$

**常用性质**：

$$\binom{n}{0} = \binom{n}{n} = 1$$
$$\binom{n}{r} = \binom{n}{n-r}$$
$$\binom{n}{r} = \binom{n-1}{r-1} + \binom{n-1}{r} \quad \text{（帕斯卡恒等式）}$$
$$2^n = \sum_{k=0}^{n} \binom{n}{k} \quad \text{（子集总数）}$$

**可重复组合**（星棒问题）：

从 n 种物品中选 r 个，允许重复：

$$\binom{n+r-1}{r}$$

### 详细讲解

**什么时候用排列？什么时候用组合？**

> **判断标准**："交换位置后，结果是否变了？"
> - 变了 → 排列（如：选班长和副班长）
> - 没变 → 组合（如：选两个代表）

**经典易错题**：

**思路**：把 AB 捆绑当作一个人，5! 种排列 × 内部 2! 种 = 240 种

```python
排列组合——使用 math 和 itertools 模块

面试中可以这样快速验证手算结果
"""

import math
from itertools import permutations, combinations, combinations_with_replacement
from typing import List


def nPr(n: int, r: int) -> int:
    """排列数 P(n,r) = n!/(n-r)!"""
    return math.perm(n, r)  # Python 3.8+


def nCr(n: int, r: int) -> int:
    """组合数 C(n,r) = n!/(r!(n-r)!)"""
    return math.comb(n, r)  # Python 3.8+


def stars_and_bars(n_types: int, r_items: int) -> int:
    """
    星棒问题（可重复组合）
    从 n 种物品中选 r 个（允许重复），方案数 = C(n+r-1, r)

    例：3 种冰淇淋选 5 球 → C(3+5-1, 5) = C(7,5) = 21
    """
    return nCr(n_types + r_items - 1, r_items)


def dice_sum_probability(n_dice: int, target: int) -> float:
    """
    掷 n 个骰子，点数和为 target 的概率

    面试经典：掷 3 个骰子，和为 10 的概率？

    方法：统计所有可能的结果数（等可能），然后用排列组合计数
    """
    # 所有可能的结果数
    total_outcomes = 6 ** n_dice

    # 枚举所有结果（仅适用于 n_dice 较小的情况）
    from itertools import product
    favorable = sum(1 for roll in product(range(1, 7), repeat=n_dice)
                    if sum(roll) == target)

    return favorable / total_outcomes
```

## 2.4 概率统计

### 概念解释

量化交易的核心数学基础。本节覆盖：随机变量、期望、方差、常见分布、大数定律、中心极限定理。

### 核心公式

**期望**：

$$E[X] = \sum_i x_i \cdot p_i \quad \text{（离散）}$$

$$E[X] = \int_{-\infty}^{\infty} x \cdot f(x) dx \quad \text{（连续）}$$

**方差**：

$$Var(X) = E[(X - \mu)^2] = E[X^2] - (E[X])^2$$

**协方差**：

$$Cov(X, Y) = E[(X - \mu_X)(Y - \mu_Y)] = E[XY] - E[X]E[Y]$$

**相关系数**：

$$\rho_{XY} = \frac{Cov(X, Y)}{\sigma_X \cdot \sigma_Y}, \quad -1 \leq \rho \leq 1$$

**夏普比率**（量化面试必考）：

$$Sharpe = \frac{R_p - R_f}{\sigma_p}$$

其中 R_p 为组合收益率，R_f 为无风险利率，σ_p 为组合波动率。

**组合方差**（两只股票）：

$$\sigma_p^2 = w_1^2\sigma_1^2 + w_2^2\sigma_2^2 + 2w_1w_2\rho\sigma_1\sigma_2$$

**二项分布**（n 次独立伯努利试验）：

$$P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}$$

**正态分布**：

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}} e^{-\frac{(x-\mu)^2}{2\sigma^2}}$$

**68-95-99.7 法则**：正态分布下，数据落在 μ±1σ 内的概率 ≈ 68%，μ±2σ ≈ 95%，μ±3σ ≈ 99.7%。

### 详细讲解

**为什么量化策略需要关注协方差和相关性？**

> 量化交易中，低相关性资产之间的组合可以**在不降低收益的情况下降低整体风险**。这就是**分散化投资**的数学原理：
>
> 如果 ρ < 1，则 σ_p < w₁σ₁ + w₂σ₂（组合风险低于加权平均风险）。

**大数定律 vs 中心极限定理**：

| 定律 | 内容 | 量化含义 |
|------|------|----------|
| 大数定律 | 样本均值 → 总体均值（n→∞） | 交易次数够多，收益趋近期望 |
| 中心极限定理 | 样本均值分布 → 正态分布（n→∞） | 可以用正态分布近似策略收益分布 |

### 🐍 Python 实战：概率统计计算（纯 Python 版——面试手写用）

```python
"""
概率统计 Python 实现（纯 Python 版本）
面试手写代码时使用——简洁、无外部依赖

资深注意：这个版本有意避免使用 numpy，因为面试白板编程
只能使用标准库。但实际量化工作中一切都要向量化。
"""

import math
from typing import Sequence


def expectation(values: Sequence[float],
                probs: Sequence[float] = None) -> float:
    """期望值 E[X] = Σ p_i · x_i"""
    if probs is None:
        return sum(values) / len(values)
    return sum(p * x for p, x in zip(probs, values))


def variance(values: Sequence[float],
             probs: Sequence[float] = None) -> float:
    """
    方差 Var(X) = E[X²] - (E[X])²

    面试要点：为什么用 E[X²] - E[X]² 而不是直接算 Σ(x-μ)²/n？
    → 数值稳定性更好（避免 catastrophic cancellation）
    如果先算均值再减，当 x 很大而方差很小时会丢失精度
    """
    if probs is None:
        n = len(values)
        # Welford 在线算法：比两步法数值更稳定
        mean = 0.0
        m2 = 0.0
        for i, x in enumerate(values, 1):
            delta = x - mean
            mean += delta / i
            m2 += delta * (x - mean)
        return m2 / n  # 总体方差
    mu = expectation(values, probs)
    return sum(p * (x - mu) ** 2 for p, x in zip(probs, values))  # noqa: F841


def std_dev(values: Sequence[float],
            probs: Sequence[float] = None) -> float:
    """标准差 σ = √Var(X)"""
    return math.sqrt(variance(values, probs))


def covariance(xs: Sequence[float], ys: Sequence[float]) -> float:
    """
    协方差 Cov(X,Y) = E[XY] - E[X]E[Y]

    资深面试点：协方差矩阵必须是半正定的（positive semi-definite）
    如果面试官给你一个相关系数矩阵不是 PSD，那是个陷阱题
    """
    n = len(xs)
    mu_x = sum(xs) / n
    mu_y = sum(ys) / n
    return sum((x - mu_x) * (y - mu_y) for x, y in zip(xs, ys)) / n


def correlation(xs: Sequence[float], ys: Sequence[float]) -> float:
    """
    皮尔逊相关系数 ρ = Cov(X,Y) / (σ_X · σ_Y)

    关键性质（面试常被追问）：
    - ρ ∈ [-1, 1]，由柯西-施瓦茨不等式保证
    - ρ = 1  ⟺ Y = aX + b (a>0)，完全正线性相关
    - ρ = -1 ⟺ Y = aX + b (a<0)，完全负线性相关
    - ρ 只度量线性相关性！ρ=0 ≠ 独立（如 Y=X²）
    """
    cov = covariance(xs, ys)
    sx = std_dev(xs)
    sy = std_dev(ys)
    return cov / (sx * sy)


def sharpe_ratio(returns: Sequence[float],
                 risk_free: float = 0.0) -> float:
    """
    夏普比率（非年化）

    Sharpe = (R̄_p - R_f) / σ_p

    资深考点：面试官可能追问——
    Q: 这个 Sharpe 是年化的吗？怎么年化？
    A: 日 Sharpe × √252（假设独立同分布）
    Q: 什么时候 Sharpe 会失效？
    A: 非正态分布（肥尾/偏态）时 Sharpe 低估风险
    """
    excess = [r - risk_free for r in returns]
    mu = expectation(excess)
    sigma = std_dev(excess)
    return mu / sigma if sigma > 0 else 0.0


def annualized_sharpe(daily_returns: Sequence[float],
                      trading_days: int = 252,
                      risk_free_annual: float = 0.0) -> float:
    """
    年化夏普比率

    Sharpe_annual = Sharpe_daily × √252

    前提假设：收益率序列独立同分布（i.i.d.）
    如果不满足（有自相关），需要用 Newey-West 修正
    """
    rf_daily = (1 + risk_free_annual) ** (1 / trading_days) - 1
    sharpe_daily = sharpe_ratio(daily_returns, rf_daily)
    return sharpe_daily * math.sqrt(trading_days)


def portfolio_variance(
    w1: float, sigma1: float,
    w2: float, sigma2: float,
    rho: float
) -> float:
    """
    两资产组合方差

    σ_p² = w₁²σ₁² + w₂²σ₂² + 2·w₁·w₂·ρ·σ₁·σ₂

    推导思路（资深面试可能要求你当场推）：
    σ_p² = Var(w₁R₁ + w₂R₂)
         = w₁²Var(R₁) + w₂²Var(R₂) + 2w₁w₂Cov(R₁,R₂)
         = w₁²σ₁² + w₂²σ₂² + 2w₁w₂ρσ₁σ₂
    """
    return (w1**2 * sigma1**2 +
            w2**2 * sigma2**2 +
            2 * w1 * w2 * rho * sigma1 * sigma2)


def binomial_prob(n: int, k: int, p: float) -> float:
    """
    二项分布 P(X=k) = C(n,k) · p^k · (1-p)^(n-k)

    数值稳定性改进（对大的 n 很重要）：
    用 log-sum-exp 避免中间溢出
    """
    # 简单版（n<1000 时可接受）
    if n <= 1000:
        return math.comb(n, k) * (p ** k) * ((1 - p) ** (n - k))

    # log 版本（大 n 时数值稳定）
    log_prob = (math.log(math.comb(n, k)) +
                k * math.log(p) +
                (n - k) * math.log(1 - p))
    return math.exp(log_prob)


def normal_pdf(x: float, mu: float = 0, sigma: float = 1) -> float:
    """正态分布概率密度函数"""
    coeff = 1 / (sigma * math.sqrt(2 * math.pi))
    exponent = -((x - mu) ** 2) / (2 * sigma ** 2)
    return coeff * math.exp(exponent)


def z_score(x: float, mu: float, sigma: float) -> float:
    """
    标准分数 z = (x - μ) / σ

    资深面试点：z-score 是标准化的核心工具，但仅当数据
    近似正态时有意义。对于肥尾分布（金融数据常见），
    用 z-score 判断"异常值"会产生大量假阳性。
    """
    return (x - mu) / sigma

print("=" * 50)
```

### 🔥 资深进阶：numpy 向量化 + 统计推断 + 数值稳定性

```python
"""
资深量化面试要求：用 numpy/scipy 进行向量化计算
核心差异：纯 Python 循环 O(n) vs numpy 向量化 O(1) 的 Python 层开销

金融数据通常百万级行数，向量化是必须的
"""

import numpy as np
from scipy import stats
from typing import Tuple


# ============ 向量化版基础统计（与实际量化工作一致） ============

def expect_vectorized(returns: np.ndarray,
                      weights: np.ndarray = None) -> float:
    """
    向量化期望值计算

    numpy 的底层 C 实现比 Python 循环快 10-100x
    """
    if weights is None:
        return float(np.mean(returns))
    return float(np.average(returns, weights=weights))


def sharpe_vectorized(returns: np.ndarray,
                      rf: float = 0.0,
                      annualize: bool = True,
                      periods: int = 252) -> float:
    """
    向量化夏普比率（年化）

    Args:
        returns: 日收益率序列
        rf: 年化无风险利率
        annualize: 是否年化
        periods: 年化周期数（日=252，周=52，月=12）

    资深考点：
    - 为什么乘以 √252 而不是 252？因为方差线性叠加（独立假设下），
      标准差是平方根关系
    - 如果收益率有自相关，√252 法则不成立，需要 Newey-West 调整
    """
    excess = returns - rf / periods
    sharpe_daily = float(np.mean(excess) / np.std(excess, ddof=1))
    if annualize:
        return sharpe_daily * np.sqrt(periods)
    return sharpe_daily


def max_drawdown(returns: np.ndarray) -> Tuple[float, int, int]:
    """
    最大回撤（向量化计算，O(n)）

    资深考点：MaxDD 是最常用的尾部风险指标之一
    面试官可能追问：MaxDD 的期望值是多少？（没有闭式解，需要模拟）
    """
    cumulative = np.cumprod(1 + returns)
    running_max = np.maximum.accumulate(cumulative)
    drawdowns = (cumulative - running_max) / running_max

    max_dd = float(np.min(drawdowns))
    end_idx = int(np.argmin(drawdowns))
    start_idx = int(np.argmax(cumulative[:end_idx + 1]))

    return max_dd, start_idx, end_idx


def portfolio_stats_vectorized(
    weights: np.ndarray,
    returns: np.ndarray,  # shape (n_periods, n_assets)
    rf: float = 0.0
) -> dict:
    """
    多资产组合统计（向量化）

    Args:
        weights: 各资产权重 (n_assets,)
        returns: 各资产收益率矩阵 (n_periods, n_assets)

    Returns:
        组合年化收益、波动率、夏普比率

    推导：
    μ_p = w^T μ           (组合期望收益)
    σ_p² = w^T Σ w         (组合方差，Σ 是协方差矩阵)
    """
    # 各资产统计
    mu = np.mean(returns, axis=0)        # (n_assets,)
    cov_matrix = np.cov(returns, rowvar=False)  # (n_assets, n_assets)

    # 组合统计（矩阵乘法，一次完成）
    port_return = float(np.dot(weights, mu))           # w^T μ
    port_var = float(weights @ cov_matrix @ weights)    # w^T Σ w
    port_vol = np.sqrt(port_var)

    # 年化（假设日收益率）
    ann_return = port_return * 252
    ann_vol = port_vol * np.sqrt(252)
    ann_sharpe = (ann_return - rf) / ann_vol if ann_vol > 0 else 0.0

    return {
        "daily_return": port_return,
        "daily_vol": port_vol,
        "ann_return": ann_return,
        "ann_vol": ann_vol,
        "ann_sharpe": ann_sharpe,
        "cov_matrix": cov_matrix,
    }


# ============ 数值稳定性 ============

def log_sum_exp(x: np.ndarray) -> float:
    """
    Log-Sum-Exp 技巧（数值稳定的 softmax/logsumexp）

    问题：直接算 Σexp(x_i) 在 x_i 很大时会溢出

    技巧：log Σexp(x_i) = c + log Σexp(x_i - c)
    其中 c = max(x_i)，将所有指数项平移到安全范围

    量化应用：处理极小的概率值、似然函数优化
    """
    c = np.max(x)
    return float(c + np.log(np.sum(np.exp(x - c))))


def stable_softmax(x: np.ndarray) -> np.ndarray:
    """数值稳定的 softmax（面试高频手写题）"""
    x_shifted = x - np.max(x)  # 防止 exp 溢出
    exp_x = np.exp(x_shifted)
    return exp_x / np.sum(exp_x)


# ============ 统计推断（资深必考） ============

def hypothesis_test_sharpe(
    returns: np.ndarray,
    benchmark_sharpe: float = 0.0,
    alpha: float = 0.05
) -> dict:
    """
    夏普比率的假设检验

    H0: Sharpe ≤ benchmark_sharpe（策略无超额收益）
    H1: Sharpe > benchmark_sharpe

    使用 Jobson & Korkie (1981) 的渐近分布：
    夏普比率的估计量近似服从 N(SR, (1+SR²/2)/n)

    资深面试点：这是判断策略是否"statistically significant"的标准方法
    """
    n = len(returns)
    sr = sharpe_vectorized(returns, annualize=False)

    # 夏普比率的标准误（Jobson-Korkie）
    se = np.sqrt((1 + sr**2 / 2) / n)

    # z 检验
    z_stat = (sr - benchmark_sharpe) / se
    p_value = 1 - stats.norm.cdf(z_stat)

    return {
        "sharpe_ratio": sr,
        "z_statistic": z_stat,
        "p_value": p_value,
        "significant": p_value < alpha,
        "se": se,
    }


def bootstrap_var(
    returns: np.ndarray,
    confidence: float = 0.95,
    n_bootstrap: int = 10000
) -> dict:
    """
    Bootstrap 法计算 VaR（Value at Risk）

    为什么不用参数法（假设正态）直接算？
    → 金融收益率通常是肥尾的（峰度 > 3）
    → 参数法会严重低估极端风险
    → Bootstrap 不需要分布假设，对肥尾更稳健

    面试中如果说"直接用 1.65σ"，资深面试官会追问"收益率是正态吗？"
    """
    alpha = 1 - confidence
    n = len(returns)

    # Bootstrap 重采样
    rng = np.random.default_rng(42)
    bootstrap_samples = rng.choice(returns, size=(n_bootstrap, n), replace=True)

    # 计算每次重采样的 VaR
    bootstrap_vars = np.percentile(bootstrap_samples, alpha * 100, axis=1)

    # 点估计和置信区间
    var_estimate = float(np.mean(bootstrap_vars))
    var_ci_lower = float(np.percentile(bootstrap_vars, 2.5))
    var_ci_upper = float(np.percentile(bootstrap_vars, 97.5))

    return {
        "var": var_estimate,
        "var_ci": (var_ci_lower, var_ci_upper),
        "confidence": confidence,
        "method": "bootstrap (non-parametric)",
    }


def cvar(returns: np.ndarray, confidence: float = 0.95) -> float:
    """
    CVaR（Conditional VaR / Expected Shortfall）

    CVaR = E[Loss | Loss > VaR]

    为什么 CVaR 比 VaR 更好？
    1. CVaR 是 coherent risk measure（满足次可加性），VaR 不满足
    2. CVaR 考虑了尾部所有损失，而不仅仅是分位点
    3. 巴塞尔 III 已用 CVaR（ES）替代 VaR

    这个知识点在面量化风控岗时几乎必问
    """
    alpha = 1 - confidence
    var_threshold = float(np.percentile(returns, alpha * 100))
    tail_losses = returns[returns <= var_threshold]
    return float(np.mean(tail_losses))


def maximum_likelihood_normal(returns: np.ndarray) -> Tuple[float, float]:
    """
    正态分布参数的最大似然估计（MLE）

    MLE 原理：选择参数 θ 使观测数据出现的概率最大
    θ* = argmax_θ ∏ f(x_i | θ)
       = argmax_θ Σ log f(x_i | θ)  （取 log 防止下溢）

    对于正态分布 N(μ, σ²)，MLE 的闭式解：
    μ̂ = Σx_i / n （样本均值）
    σ̂² = Σ(x_i - μ̂)² / n （有偏的 MLE，分母是 n 不是 n-1）

    资深考点：为什么 MLE 方差分母是 n 而不是 n-1？
    → MLE 是无偏性不一定保证。σ̂²_MLE 是有偏但一致的
    → 样本方差（分母 n-1）才是无偏估计
    """
    n = len(returns)
    mu_mle = float(np.mean(returns))
    sigma_mle = float(np.std(returns))  # numpy 默认 ddof=0 = MLE

    # 对数似然
    log_likelihood = float(
        -n/2 * np.log(2 * np.pi)
        - n/2 * np.log(sigma_mle**2)
        - np.sum((returns - mu_mle)**2) / (2 * sigma_mle**2)
    )

    return mu_mle, sigma_mle
```

## 2.5 等差数列

### 概念解释

相邻两项的差为常数的数列。用于等额定投、线性摊销、简单利息等金融场景。

### 核心公式

**第 n 项**：

$$a_n = a_1 + (n-1)d$$

**前 n 项和**：

$$S_n = \frac{n(a_1 + a_n)}{2} = \frac{n[2a_1 + (n-1)d]}{2}$$

**等差中项**：若 a, b, c 成等差，则 b = (a+c)/2。

### 🐍 Python 实战：等差数列

```python
"""
等差数列——Python 实现
"""

from typing import List


def arithmetic_nth(a1: float, n: int, d: float) -> float:
    """第 n 项：a_n = a1 + (n-1)·d"""
    return a1 + (n - 1) * d


def arithmetic_sum(a1: float, n: int, d: float) -> float:
    """前 n 项和：S_n = n·(a1 + a_n) / 2"""
    an = arithmetic_nth(a1, n, d)
    return n * (a1 + an) / 2


def arithmetic_sequence_list(a1: float, d: float,
                              n_terms: int) -> List[float]:
    """生成等差数列前 n 项"""
    return [a1 + i * d for i in range(n_terms)]


# --- 演示：定投份额计算 ---
def dca_analysis(monthly_invest: float = 1000,
                 months: int = 12,
                 nav_first: float = 1.00,
                 nav_drop: float = 0.02):
    """
    等额定投分析（Dollar Cost Averaging）

    每月定投固定金额，净值线性下降（等差），
    计算总份额和平均成本
    """
    navs = [nav_first - i * nav_drop for i in range(months)]

    total_invested = monthly_invest * months
    total_shares = sum(monthly_invest / nav for nav in navs)
    avg_cost = total_invested / total_shares
    arithmetic_avg_nav = sum(navs) / len(navs)

    print(f"\n定投分析 ({months}个月，每月{monthly_invest}元)：")
    print(f"  净值序列: {[f'{nav:.2f}' for nav in navs]}")
    print(f"  总投入: {total_invested:,.0f} 元")
    print(f"  总份额: {total_shares:,.0f} 份")
    print(f"  实际平均成本: {avg_cost:.4f} 元/份")
    print(f"  算术平均净值: {arithmetic_avg_nav:.4f} 元/份")
    print(f"  摊薄效应: 实际成本比算术均值低 "
          f"{(arithmetic_avg_nav - avg_cost):.4f} 元")
```

## 2.6 等比数列

### 概念解释

相邻两项的比为常数的数列。用于复利计算、几何平均收益率、对数收益等金融场景。

### 核心公式

**第 n 项**：

$$a_n = a_1 \cdot q^{n-1}$$

**前 n 项和**（q ≠ 1）：

$$S_n = \frac{a_1(1 - q^n)}{1 - q}$$

**无穷等比级数和**（|q| < 1）：

$$S_{\infty} = \frac{a_1}{1 - q}$$

### 🐍 Python 实战：等比数列与复利

```python
"""
等比数列——Python 实现
用于复利计算、几何增长等金融场景
"""

import math
from typing import List


def geometric_nth(a1: float, n: int, q: float) -> float:
    """第 n 项：a_n = a1 · q^(n-1)"""
    return a1 * (q ** (n - 1))


def geometric_sum(a1: float, n: int, q: float) -> float:
    """前 n 项和：S_n = a1·(1-q^n) / (1-q)  (q≠1)"""
    if q == 1:
        return a1 * n
    return a1 * (1 - q ** n) / (1 - q)


def geometric_infinite_sum(a1: float, q: float) -> float:
    """无穷等比级数和：S_∞ = a1 / (1-q)  (|q| < 1)"""
    if abs(q) >= 1:
        raise ValueError("无穷级数收敛要求 |q| < 1")
    return a1 / (1 - q)


def compound_growth(principal: float, annual_rate: float,
                    years: int) -> float:
    """复利终值：P · (1+r)^n"""
    return principal * ((1 + annual_rate) ** years)


def years_to_double(annual_rate: float) -> float:
    """计算本金翻倍需要的年数：n = ln(2) / ln(1+r)"""
    return math.log(2) / math.log(1 + annual_rate)


def daily_compound_annual_return(daily_return: float,
                                 trading_days: int = 250) -> float:
    """
    日复利 → 年化收益率

    年化 = (1 + r_daily)^250 - 1

    例：日收益 0.1% → 年化 ≈ 28.4%
    """
    return (1 + daily_return) ** trading_days - 1

print(f"  等比第5项 (a1=3, q=2): {geometric_nth(3, 5, 2)}")
print(f"  等比前5项和: {geometric_sum(3, 5, 2)}")
print(f"  无穷级数 (a1=1, q=0.5): {geometric_infinite_sum(1, 0.5)}")
print()

# 复利演示
principal = 100000
rate = 0.08
years = 10
future = compound_growth(principal, rate, years)
print(f"复利: {principal:,.0f} 元，年化 {rate:.0%}，{years} 年后 = "
      f"{future:,.0f} 元")

# 翻倍年数
for r in [0.06, 0.08, 0.10, 0.15]:
    ny = years_to_double(r)
    rule72 = 72 / (r * 100)
    print(f"  年化 {r:.0%} → 翻倍需 {ny:.2f} 年 (72法则: {rule72:.1f} 年)")

# 复利的威力
daily = 0.001  # 0.1% 每天
ann = daily_compound_annual_return(daily)
```

## 3.1 做市商定价（期望值为零的博弈）

做市商报出买卖价差（bid-ask spread），通过统计优势盈利。

**核心逻辑**：做市商不需要预测方向，只需要**买卖价差 > 逆向选择成本**。

**考题**：你做市某股票，bid 99.90，ask 100.10。有 55% 的概率下一个交易者是"噪音交易者"（随机买卖），45% 的概率是"知情交易者"（知道方向）。知情交易者买入时股票会上涨 0.5%，卖出时下跌 0.5%。你的预期利润是多少？

<details>
<summary>点击查看答案</summary>

分析每一笔 incoming trade：

**情况1：噪音交易者（55%）**
- 随机买或卖，期望盈亏为 0（对称）

**情况2：知情交易者（45%）**
- 他知道方向，当你报 ask 100.10 时他只在"股票被低估"时才买 → 你亏
- 当你报 bid 99.90 时他只在"股票被高估"时才卖 → 你亏
- 每次遇到知情交易者，你期望亏损 = 价差的"不利选择"部分

简化版：每笔交易的期望 = 0.55 × (价差/2) + 0.45 × (-逆向选择损失)

如果价差 0.20，知情者带来的期望损失为 0.15 元/笔，则：
期望利润 = 0.55 × 0.10 + 0.45 × (-0.15) = 0.055 - 0.0675 = **-0.0125**

这意味着需要**扩大价差**或**提高噪音交易者比例**才能盈利。
</details>

## 3.2 凯利公式（下注比例）

**凯利公式**决定每次下注的最优资金比例，使长期增长率最大化：

$$f^* = \frac{p \cdot b - (1-p)}{b} = \frac{p(b+1) - 1}{b}$$

其中：p = 胜率，b = 赔率（净赢/输，即盈亏比）。

**简化版（赢则赢1倍，输则输光）**：

$$f^* = 2p - 1$$

### 🐍 Python 实战：凯利公式与仓位管理

```python
"""
凯利公式（Kelly Criterion）Python 实现

量化面试高频考点：如何确定最优下注比例

核心公式：f* = (p·b - (1-p)) / b
  p: 胜率
  b: 盈亏比（净赢/净输）

半凯利：f*/2，用于控制回撤
"""


def kelly_fraction(win_prob: float, win_loss_ratio: float) -> float:
    """
    凯利公式：计算最优下注比例

    Args:
        win_prob: 胜率 p (0~1)
        win_loss_ratio: 盈亏比 b = (盈利金额) / (亏损金额)

    Returns:
        最优下注比例 f* (0~1)
        负值表示不应下注

    Example:
        >>> kelly_fraction(0.6, 2.0)   # 60%胜率, 2:1盈亏比
        0.4
        >>> kelly_fraction(0.6, 0.5)   # 60%胜率, 0.5:1盈亏比
        -0.2  # 不应参与！
    """
    return (win_prob * win_loss_ratio - (1 - win_prob)) / win_loss_ratio


def half_kelly(win_prob: float, win_loss_ratio: float) -> float:
    """
    半凯利公式：f*/2

    实践中更常用的保守策略，降低回撤风险
    虽然长期增长率略低于全凯利，但波动率大幅降低
    """
    return kelly_fraction(win_prob, win_loss_ratio) / 2


def kelly_simplified(win_prob: float) -> float:
    """
    简化版凯利公式（盈亏比=1，即赢则翻倍、输则输光）

    f* = 2p - 1
    只有当 p > 0.5 时才应该参与
    """
    return 2 * win_prob - 1


def simulate_kelly_betting(
    initial_capital: float = 10000,
    n_bets: int = 100,
    win_prob: float = 0.6,
    win_loss_ratio: float = 1.5,
    kelly_fraction_used: float = None
) -> dict:
    """
    模拟凯利策略的长期表现

    对比：全凯利 vs 半凯利 vs 固定比例 vs 全仓
    """
    import random

    if kelly_fraction_used is None:
        kelly_fraction_used = kelly_fraction(win_prob, win_loss_ratio)

    strategies = {
        "全凯利": kelly_fraction_used,
        "半凯利": kelly_fraction_used / 2,
        "25%仓位": 0.25,
        "全仓ALL-IN": 1.0,
    }

    results = {name: [initial_capital] for name in strategies}

    for _ in range(n_bets):
        win = random.random() < win_prob
        for name, fraction in strategies.items():
            prev = results[name][-1]
            bet_amount = prev * fraction
            if win:
                prev += bet_amount * win_loss_ratio  # 赢：赚 b 倍下注额
            else:
                prev -= bet_amount                    # 输：亏掉下注额
            results[name].append(prev)

    return results

# 场景：胜率 60%，每次盈利 2%，亏损 1.5%

b = 0.02 / 0.015  # = 1.333
f_star = kelly_fraction(p, b)

print(f"\n  策略参数：胜率={p:.0%}, 盈亏比={b:.2f}:1")
print(f"  全凯利最优仓位: {f_star:.1%}")
print(f"  半凯利仓位:     {half_kelly(p, b):.1%}")
print(f"  简化凯利(p=0.6): {kelly_simplified(0.6):.1%}")

# 检查是否值得参与
if f_star <= 0:
    print("  ⚠️  凯利公式给出负值：此策略不值得参与！")
else:
    print(f"  ✅ 正期望策略，每次投入 {f_star:.1%} 仓位最优")

print()

# 不同胜负率下的凯利比例
print("  不同胜率下的最优仓位（盈亏比 = 2:1）：")
for p_test in [0.3, 0.4, 0.5, 0.55, 0.6, 0.7]:
    f = kelly_fraction(p_test, 2.0)
    print(f"    胜率 {p_test:.0%} → f* = {f:+.1%}  "
          f"{'← 不参与' if f <= 0 else ''}")
```

**考题**：一个策略胜率 60%，每次盈利 2%，亏损 1.5%。每次应投入多少仓位？

<details>
<summary>点击查看答案</summary>

b = 2% / 1.5% = 1.333

$$f^* = \frac{0.6 \times 1.333 - 0.4}{1.333} = \frac{0.8 - 0.4}{1.333} = \frac{0.4}{1.333} = 0.30$$

即每次投入 **30%** 仓位。

**实际注意**：凯利公式给出理论最优值，实践中通常使用**半凯利**（f*/2 = 15%）来控制回撤。
</details>

### 🔥 资深面试策略：凯利公式的深度追问

**追问1**：为什么不直接用全凯利？
→ 全凯利虽然最大化长期增长率，但：
  - 波动极大（方差也是最大的稳定策略）
  - 对 p 和 b 的估计误差极其敏感。如果高估了 p，全凯利可能让你破产
  - 实践中 1/2 甚至 1/4 凯利更常见

**追问2**：如果 p 和 b 都是估计值（有误差），你怎么调整仓位？
→ 使用 **Fractional Kelly** 或 **Bayesian Kelly**：
  - 对 p 使用 Beta 分布（共轭先验），取其下分位数而非点估计
  - 加一个"不确定性折价"：f*_robust = f*(p̂ - 1.96·SE, b̂)，代入参数估计最坏情况

**追问3**：你有多个独立策略同时运行，怎么用凯利公式分配总仓位？
→ 多策略凯利：
  - 如果策略收益率独立：各策略各用凯利，然后**加总**（因为独立策略的仓位可以叠加）
  - 如果策略相关：需要**联合凯利**，f* = Σ^{-1} · edge（Σ 是策略收益率的协方差矩阵）

```python
def robust_kelly(win_prob_est: float, win_loss_ratio: float,
                 prob_uncertainty: float = 0.05) -> float:
    """
    鲁棒凯利：考虑胜率估计不确定性的保守仓位

    用胜率估计的下边界（95% CI 的下限）而非点估计
    这对新手面试官展示风险管理意识非常有效
    """
    p_conservative = max(0, win_prob_est - prob_uncertainty)
    return kelly_fraction(p_conservative, win_loss_ratio)

def multi_strategy_kelly(edges: np.ndarray,
                          cov: np.ndarray) -> np.ndarray:
    """
    多策略联合凯利（需要相关矩阵）

    f* = Σ^{-1} · edge

    其中 edge_i = P(win_i)·b_i - (1-P(win_i))
    当策略完全独立时，退化为各策略各用凯利
    """
    # 用 numpy 求解线性系统而非直接求逆（更稳定）
    return np.linalg.solve(cov, edges)
```

## 3.3 停时问题与最优停止

**经典问题——秘书问题/相亲问题**：

> N 个候选人按随机顺序出现，你每次只能选择接受或拒绝（不能回头）。如何最大化选中最好的那个人的概率？

**最优策略**：跳过前 1/e ≈ 37% 的候选人（仅观察不选），之后遇到比之前所有都好的就选。

**量化应用**：最优止损、最优建仓时机、回撤阈值设定。

### 🐍 Python 实战：最优停止问题模拟

```python
"""
秘书问题（最优停止问题）
在不确定序列中最大化选中最优的概率

结论：前 1/e ≈ 37% 只观察不选，之后遇到更好的就选
"""

import random
import math


def secretary_simulation(n_candidates: int = 100,
                         n_trials: int = 10000,
                         look_fraction: float = 1/math.e) -> dict:
    """
    蒙特卡洛模拟秘书问题

    Args:
        n_candidates: 候选人数量
        n_trials: 模拟次数
        look_fraction: 观察比例（默认 1/e ≈ 37%）

    Returns:
        统计结果字典
    """
    look_until = int(n_candidates * look_fraction)
    success = 0

    for _ in range(n_trials):
        # 随机排列候选人质量
        candidates = list(range(1, n_candidates + 1))
        random.shuffle(candidates)

        # 前 look_until 个只观察，记录最高分
        best_so_far = max(candidates[:look_until]) if look_until > 0 else 0

        # 之后遇到更好的就选
        chosen = None
        for c in candidates[look_until:]:
            if c > best_so_far:
                chosen = c
                break

        # 如果没找到更好的，被迫选最后一个
        if chosen is None:
            chosen = candidates[-1]

        # 检查是否选到了最优（即 n_candidates）
        if chosen == n_candidates:
            success += 1

    return {
        "n_candidates": n_candidates,
        "n_trials": n_trials,
        "look_fraction": look_fraction,
        "look_until": look_until,
        "success_rate": success / n_trials,
        "theoretical_best": 1 / math.e,  # ≈ 0.3679
    }


# 演示
result = secretary_simulation(n_candidates=100, n_trials=5000)
print(f"最优停止（秘书问题）模拟：")
print(f"  前 {result['look_until']} 人只观察 ({result['look_fraction']:.1%})")
print(f"  选中最优的概率: {result['success_rate']:.2%}")
print(f"  理论最优概率:   {result['theoretical_best']:.2%}")

# 注意：即使最优策略，选中最好的概率也只有约 37%
# 但这已经是最优策略了
```

## 3.4 随机游走与布朗运动

收益率建模的基础。

**随机游走**：

$$P_t = P_{t-1} + \epsilon_t, \quad \epsilon_t \sim N(0, \sigma^2)$$

**几何布朗运动**（Black-Scholes 基础）：

$$dS = \mu S dt + \sigma S dW$$

其中 dW 是维纳过程。

### 🐍 Python 实战：随机游走模拟

```python
"""
随机游走与几何布朗运动模拟
"""

import random
import math
from typing import List, Tuple


def random_walk(
    initial_price: float = 100,
    n_steps: int = 252,
    daily_vol: float = 0.01,
    drift: float = 0.0,
    seed: int = None
) -> List[float]:
    """
    模拟随机游走（无漂移或有漂移）

    P_t = P_{t-1} + ε_t,  ε_t ~ N(0, σ²)

    Args:
        initial_price: 初始价格
        n_steps: 步数（如交易天数）
        daily_vol: 日波动率 σ
        drift: 日漂移率 μ（默认 0 为无漂移）
        seed: 随机种子（用于复现）

    Returns:
        价格路径
    """
    if seed is not None:
        random.seed(seed)

    prices = [initial_price]
    for _ in range(n_steps):
        # 使用 Box-Muller 或简单近似生成正态随机数
        epsilon = random.gauss(mu=drift, sigma=daily_vol)
        prices.append(prices[-1] + epsilon)

    return prices


def geometric_brownian_motion(
    initial_price: float = 100,
    n_steps: int = 252,
    daily_vol: float = 0.01,
    daily_drift: float = 0.0005,  # 0.05% per day
    seed: int = None
) -> List[float]:
    """
    几何布朗运动（GBM）
    dS = μ·S·dt + σ·S·dW

    离散化（Euler-Maruyama）：
    S_{t+1} = S_t · exp((μ - σ²/2)·Δt + σ·√Δt·ε_t)
    """
    if seed is not None:
        random.seed(seed)

    dt = 1.0  # 每步 = 1 天
    prices = [initial_price]

    for _ in range(n_steps):
        epsilon = random.gauss(0, 1)  # 标准正态
        drift_term = (daily_drift - 0.5 * daily_vol ** 2) * dt
        diffusion_term = daily_vol * math.sqrt(dt) * epsilon
        prices.append(prices[-1] * math.exp(drift_term + diffusion_term))

    return prices


def price_range_probability(
    initial_price: float = 100,
    n_days: int = 100,
    daily_vol: float = 0.01,
    lower_pct: float = -0.10,
    upper_pct: float = 0.10,
    n_simulations: int = 10000
) -> float:
    """
    模拟未来 N 天后价格落在给定区间的概率

    例：100 元股票，100 天后在 90-110 之间的概率？
    （假设无漂移随机游走，日波动率 1%）

    理论值：
    - 100 天后方差 = n × σ² = 100 × 0.01² = 0.01
    - 标准差 = √0.01 = 0.10 = 10% × 100 = 10 元
    - 90-110 = 100 ± 10 = ±1σ → 约 68%
    """
    count_in_range = 0
    for _ in range(n_simulations):
        path = random_walk(initial_price, n_days, daily_vol)
        final_price = path[-1]
        lower = initial_price * (1 + lower_pct)
        upper = initial_price * (1 + upper_pct)
        if lower <= final_price <= upper:
            count_in_range += 1

    return count_in_range / n_simulations


# 演示
print("随机游走模拟：")
prob = price_range_probability(
    initial_price=100, n_days=100, daily_vol=0.01,
    lower_pct=-0.10, upper_pct=0.10, n_simulations=10000
)
print(f"  100天后价格在 90-110 之间的概率: {prob:.1%} (理论 ≈ 68%)")

# GBM 演示
gbm_path = geometric_brownian_motion(
    initial_price=100, n_steps=5, daily_vol=0.02, daily_drift=0.0005, seed=42
)
print(f"  GBM 示例路径 (前5步): {[f'{p:.2f}' for p in gbm_path]}")
```

## 3.5 🔥 资深进阶：马科维茨组合优化

### 概念解释

马科维茨（Markowitz）均值-方差优化是现代组合理论的基石。资深量化岗几乎必问。

**核心问题**：给定 N 个风险资产，在目标收益约束下，最小化组合风险。

### 数学推导

**优化问题（最小方差组合）**：

$$\min_{\mathbf{w}} \mathbf{w}^T \Sigma \mathbf{w} \quad \text{s.t.} \quad \mathbf{w}^T \mathbf{1} = 1$$

**带收益目标（有效前沿上的点）**：

$$\min_{\mathbf{w}} \mathbf{w}^T \Sigma \mathbf{w} \quad \text{s.t.} \quad \mathbf{w}^T \boldsymbol{\mu} = \mu_{\text{target}},\ \mathbf{w}^T \mathbf{1} = 1$$

**拉格朗日乘子法求解**（闭式解）：

拉格朗日函数：$$\mathcal{L} = \frac{1}{2}\mathbf{w}^T\Sigma\mathbf{w} - \lambda(\mathbf{w}^T\boldsymbol{\mu} - \mu_t) - \gamma(\mathbf{w}^T\mathbf{1} - 1)$$

一阶条件：$$\frac{\partial\mathcal{L}}{\partial\mathbf{w}} = \Sigma\mathbf{w} - \lambda\boldsymbol{\mu} - \gamma\mathbf{1} = 0$$

解得：$$\mathbf{w}^* = \Sigma^{-1}(\lambda\boldsymbol{\mu} + \gamma\mathbf{1})$$

**最大夏普比率组合（Tangency Portfolio）**：

$$\max_{\mathbf{w}} \frac{\mathbf{w}^T\boldsymbol{\mu} - r_f}{\sqrt{\mathbf{w}^T\Sigma\mathbf{w}}}$$

### 🐍 Python 实战：完整组合优化

```python
"""
马科维茨均值-方差优化（完整实现）

资深面试重点：
1. 闭式解 vs 数值优化（带约束时）
2. 协方差矩阵估计的敏感性
3. 约束的意义（做空限制、行业集中度等）
"""

import numpy as np
from scipy.optimize import minimize
from typing import Tuple, Optional


# ============ 数据准备 ============

def estimate_parameters(returns: np.ndarray) -> Tuple[np.ndarray, np.ndarray]:
    """
    从历史收益率估计 μ 和 Σ

    资深经验：
    - 样本协方差矩阵对异常值敏感，实践中常用 shrinkage estimator（Ledoit-Wolf）
    - 期望收益 μ 的估计极不可靠（"mean-blur"问题），是组合优化最大的痛点
    - 面试时提这一点会给面试官留下深刻印象
    """
    mu = np.mean(returns, axis=0).reshape(-1, 1)  # (n,) → (n,1)
    cov = np.cov(returns, rowvar=False)
    return mu, cov


# ============ 闭式解（无约束） ============

def min_variance_portfolio(cov: np.ndarray) -> np.ndarray:
    """
    全局最小方差组合（闭式解）

    问题：min w^T Σ w  s.t. w^T 1 = 1

    闭式解：w* = Σ^{-1} 1 / (1^T Σ^{-1} 1)

    推导：拉格朗日乘子法
    L = (1/2) w^T Σ w - λ(w^T 1 - 1)
    ∂L/∂w = Σ w - λ·1 = 0 → w ∝ Σ^{-1} 1
    约束 w^T 1 = 1 → λ = 1 / (1^T Σ^{-1} 1)
    """
    inv_cov = np.linalg.inv(cov)  # 注意：实际中应避免显式求逆
    ones = np.ones((len(cov), 1))

    # 用 solve 而非 inv 更稳定（面试时可以提到）
    numerator = np.linalg.solve(cov, ones)  # Σ^{-1} 1
    denominator = ones.T @ numerator       # 1^T Σ^{-1} 1

    w = numerator / denominator
    return w.flatten()


def efficient_portfolio(
    mu: np.ndarray,
    cov: np.ndarray,
    target_return: float
) -> np.ndarray:
    """
    有效前沿上的点（给定期望收益，最小化风险）

    闭式解（两基金分离定理）：
    w* = a + b·μ_target

    其中 a, b 是常数向量，仅取决于 μ 和 Σ
    """
    inv_cov = np.linalg.inv(cov)
    ones = np.ones((len(cov), 1))

    # 标量参数
    A = float(ones.T @ inv_cov @ mu)       # 1^T Σ^{-1} μ
    B = float(mu.T @ inv_cov @ mu)          # μ^T Σ^{-1} μ
    C = float(ones.T @ inv_cov @ ones)      # 1^T Σ^{-1} 1
    D = B * C - A ** 2

    if abs(D) < 1e-12:
        raise ValueError("协方差矩阵可能是奇异的")

    # 有效前沿权重
    g = (B * inv_cov @ ones - A * inv_cov @ mu) / D
    h = (C * inv_cov @ mu - A * inv_cov @ ones) / D
    w = g + h * target_return

    return w.flatten()


def max_sharpe_portfolio(
    mu: np.ndarray,
    cov: np.ndarray,
    risk_free: float = 0.0
) -> np.ndarray:
    """
    最大夏普比率组合（Tangency Portfolio）

    闭式解：
    w* ∝ Σ^{-1} (μ - r_f·1)

    直观理解：每单位风险获得最高超额收益的组合
    这是CAPM中市场组合的理论基础
    """
    inv_cov = np.linalg.inv(cov)
    ones = np.ones((len(cov), 1))
    excess_mu = mu - risk_free  # 超额收益

    numerator = inv_cov @ excess_mu
    denominator = ones.T @ numerator

    w = numerator / denominator
    return w.flatten()


# ============ 数值优化（带约束的实际情况） ============

def portfolio_optimization_numeric(
    mu: np.ndarray,
    cov: np.ndarray,
    risk_free: float = 0.0,
    long_only: bool = True,
    max_weight: float = 1.0,
    target_return: Optional[float] = None,
) -> np.ndarray:
    """
    有约束的组合优化（scipy 数值求解）

    现实量化中几乎都有约束：
    - 做空限制（long-only）
    - 个股权重上限
    - 行业集中度限制
    - 换手率限制

    面试要点：要能解释为什么用数值优化而非闭式解
    → 因为加了不等式约束后没有闭式解
    """
    n = len(mu)

    def neg_sharpe(w):
        port_ret = w @ mu
        port_vol = np.sqrt(w @ cov @ w)
        return -(port_ret - risk_free) / port_vol

    def port_variance(w):
        return w @ cov @ w

    # 约束
    constraints = [{"type": "eq", "fun": lambda w: np.sum(w) - 1.0}]
    if target_return is not None:
        constraints.append(
            {"type": "eq", "fun": lambda w: w @ mu - target_return}
        )

    # 边界
    bounds = [(0, max_weight) if long_only else (-1, max_weight)
              for _ in range(n)]

    # 初始猜测（等权）
    w0 = np.ones(n) / n

    # 优化
    if target_return is not None:
        result = minimize(port_variance, w0, method="SLSQP",
                          bounds=bounds, constraints=constraints)
    else:
        result = minimize(neg_sharpe, w0, method="SLSQP",
                          bounds=bounds, constraints=constraints)

    return result.x


# ============ 有效前沿绘制 ============

def efficient_frontier_points(
    mu: np.ndarray,
    cov: np.ndarray,
    n_points: int = 50,
    long_only: bool = False
) -> Tuple[np.ndarray, np.ndarray]:
    """
    计算有效前沿上的点

    面试技巧：被要求"画出有效前沿"时，可以用参数化方法
    沿等收益线求解最小方差组合
    """
    min_ret = float(np.min(mu))
    max_ret = float(np.max(mu))

    if long_only:
        # 有卖空限制时用数值优化
        target_returns = np.linspace(min_ret, max_ret, n_points)
        vols = []
        for target in target_returns:
            try:
                w = portfolio_optimization_numeric(
                    mu, cov, target_return=target, long_only=True
                )
                vols.append(np.sqrt(w @ cov @ w))
            except Exception:
                vols.append(np.nan)

        valid = ~np.isnan(vols)
        return np.array(vols)[valid], target_returns[valid]
    else:
        # 无卖空限制时可以用闭式解
        min_var_w = min_variance_portfolio(cov)
        min_var_ret = float(min_var_w @ mu)
        min_var_vol = float(np.sqrt(min_var_w @ cov @ min_var_w))

        target_returns = np.linspace(min_var_ret, max_ret * 1.2, n_points)
        vols = []
        valid_rets = []

        for target in target_returns:
            try:
                w = efficient_portfolio(mu, cov, target)
                vols.append(np.sqrt(float(w @ cov @ w)))
                valid_rets.append(target)
            except Exception:
                continue

        return np.array(vols), np.array(valid_rets)

if __name__ == "__main__":
```

## 3.6 🔥 资深进阶：风险管理体系

### 概念解释

量化私募面试中，风险管理问题区分初级和资深候选人。初级能算 VaR，资深能讲清楚 VaR 的局限、CVaR 的优势、以及极端风险建模。

### 核心风险指标

| 指标 | 公式 | 优缺点 |
|------|------|--------|
| **VaR** (Value at Risk) | P(Loss > VaR) = α | 直观但不满足次可加性 |
| **CVaR** (Expected Shortfall) | E[Loss \| Loss > VaR] | 一致风险度量，巴塞尔III采用 |
| **Max Drawdown** | max(1 - V_t / peak_t) | 最直观，但无闭式分布 |
| **Calmar Ratio** | Ann_Return / MaxDD | 考虑回撤的风险调整收益 |
| **Sortino Ratio** | (R - R_f) / σ_down | 只惩罚下行波动率 |
| **Omega Ratio** | Σmax(0, R - τ)/Σmax(0, τ - R) | 考虑整个收益分布 |

### 🐍 Python 实战：完整风险管理模块

```python
"""
风险管理完整模块（资深量化必备）
"""

import numpy as np
from scipy import stats
from typing import Dict, Optional


def value_at_risk_parametric(
    returns: np.ndarray,
    confidence: float = 0.95
) -> float:
    """
    参数法 VaR（假设正态分布）

    VaR_α = -(μ + z_α · σ)

    这是"教科书"方法，也是最有问题的

    资深要点：面试时主动指出——
    1. 金融收益率肥尾（峰度 >> 3），正态假设严重低估极端风险
    2. 参数法 VaR 对分布假设极其敏感
    3. 2008 金融危机暴露了 VaR 的根本缺陷
    """
    mu = np.mean(returns)
    sigma = np.std(returns, ddof=1)
    z = stats.norm.ppf(1 - confidence)  # 95% → 1.645
    return float(-(mu + z * sigma))


def value_at_risk_historical(
    returns: np.ndarray,
    confidence: float = 0.95
) -> float:
    """
    历史模拟法 VaR（无分布假设）

    直接用历史分位数——最保守也最透明

    优点：无分布假设，捕捉了肥尾
    缺点：假设历史重复，对结构性变化不敏感
    """
    alpha = 1 - confidence
    return float(-np.percentile(returns, alpha * 100))


def value_at_risk_cornish_fisher(
    returns: np.ndarray,
    confidence: float = 0.95
) -> float:
    """
    Cornish-Fisher 展开修正 VaR（考虑偏度和峰度）

    z_cf = z_α + (z_α²-1)·S/6 + (z_α³-3z_α)·K/24 - (2z_α³-5z_α)·S²/36

    其中 S = 偏度，K = 超额峰度

    这比参数法 VaR 好得多，面试中提到可以展示统计功底
    """
    mu = np.mean(returns)
    sigma = np.std(returns, ddof=1)
    S = stats.skew(returns)       # 偏度
    K = stats.kurtosis(returns)    # 超额峰度（excess kurtosis）

    z = stats.norm.ppf(1 - confidence)

    # Cornish-Fisher 修正的分位数
    z_cf = (z
            + (z**2 - 1) * S / 6
            + (z**3 - 3*z) * K / 24
            - (2*z**3 - 5*z) * S**2 / 36)

    return float(-(mu + z_cf * sigma))


def expected_shortfall_historical(
    returns: np.ndarray,
    confidence: float = 0.95
) -> float:
    """
    历史模拟法 CVaR（Expected Shortfall）

    ES_α = -E[R | R ≤ -VaR_α]

    巴塞尔 III 已用 ES 替代 VaR 作为核心风险指标
    原因：ES 是 coherent risk measure（满足次可加性）
    """
    alpha = 1 - confidence
    var = -np.percentile(returns, alpha * 100)
    tail = returns[returns <= -var]
    return float(-np.mean(tail)) if len(tail) > 0 else var


def max_drawdown_vectorized(returns: np.ndarray) -> Dict:
    """
    最大回撤及恢复分析（向量化）

    资深面试点：
    - MaxDD 有路径依赖（path-dependent），不可能用闭式分布描述
    - 实践中用 Monte Carlo 模拟 MaxDD 的期望分布
    - 一个可以讨论的 trade-off 话题：更高的 Sharpe vs 更低的 MaxDD
    """
    cumulative = np.cumprod(1 + returns)
    peak = np.maximum.accumulate(cumulative)
    drawdown = (cumulative - peak) / peak

    # 最大回撤
    max_dd = float(np.min(drawdown))
    end = int(np.argmin(drawdown))
    start = int(np.argmax(cumulative[:end + 1]))

    # 恢复时间（从谷底回到前高的交易日数）
    trough = cumulative[end]
    recovery_idx = np.argmax(cumulative[end:] >= peak[end])
    recovery_days = recovery_idx if recovery_idx > 0 else -1  # -1 表示尚未恢复

    # 回撤期间的统计
    underwater = drawdown[drawdown < 0]

    return {
        "max_drawdown": max_dd,
        "max_dd_start": start,
        "max_dd_end": end,
        "recovery_days": recovery_days,
        "avg_drawdown": float(np.mean(underwater)) if len(underwater) > 0 else 0.0,
        "drawdown_std": float(np.std(underwater)) if len(underwater) > 0 else 0.0,
        "num_drawdowns": np.sum(
            (drawdown < 0) & (np.roll(drawdown, 1) >= 0)
        ),
    }


def risk_decomposition(
    weights: np.ndarray,
    cov_matrix: np.ndarray
) -> Dict:
    """
    风险分解（Risk Decomposition）

    组合总风险 σ_p = √(w^T Σ w)

    边际风险贡献（MRC）：∂σ_p/∂w_i = (Σw)_i / σ_p
    成分风险贡献（CRC）：w_i · MRC_i
    风险贡献百分比：CRC_i / σ_p

    面试要点：这是理解"哪个资产贡献了最多风险"的标准框架
    """
    port_sigma = float(np.sqrt(weights @ cov_matrix @ weights))

    # 边际风险贡献
    mrc = (cov_matrix @ weights) / port_sigma  # (Σw) / σ_p

    # 成分风险贡献
    crc = weights * mrc

    # 总和应等于组合波动率
    assert abs(np.sum(crc) - port_sigma) < 1e-10

    return {
        "portfolio_volatility": port_sigma,
        "marginal_risk_contributions": mrc,
        "component_risk_contributions": crc,
        "risk_contributions_pct": crc / port_sigma,
    }


def stress_test_scenarios(
    returns: np.ndarray,
    scenarios: Dict[str, float]
) -> Dict[str, float]:
    """
    压力测试：在不同历史场景下的表现

    scenarios 示例：
    {
        "COVID_2020": (date_start, date_end),
        "2015_crash": (date_start, date_end),
    }

    资深经验：压力测试 + 敏感性分析是 PM 最想看到的
    """
    results = {}
    for name, multiplier in scenarios.items():
        # 简单版：用收益率乘以压力乘数
        stressed_returns = returns * multiplier
        results[name] = {
            "cum_return": float(np.prod(1 + stressed_returns) - 1),
            "max_dd": max_drawdown_vectorized(stressed_returns)["max_drawdown"],
        }
    return results
```

## 3.7 🔥 资深进阶：随机过程与 Black-Scholes

### 概念解释

量化策略（尤其是期权/衍生品相关策略）建立在随机过程理论之上。资深面试中常涉及伊藤引理、GBM 的性质、鞅定价。

### 核心概念

**几何布朗运动（GBM）—— Black-Scholes 的基础**：

$$dS_t = \mu S_t dt + \sigma S_t dW_t$$

其中 W_t 是标准布朗运动（维纳过程）。

**伊藤引理（Ito's Lemma）—— 随机微积分的链式法则**：

对于 f(S_t, t)，有：

$$df = \left(\frac{\partial f}{\partial t} + \mu S \frac{\partial f}{\partial S} + \frac{1}{2}\sigma^2 S^2 \frac{\partial^2 f}{\partial S^2}\right)dt + \sigma S \frac{\partial f}{\partial S} dW_t$$

**GBM 的解析解**（面试必考推导）：

$$S_t = S_0 \exp\left((\mu - \frac{\sigma^2}{2})t + \sigma W_t\right)$$

**对数收益率的分布**：

$$\ln(S_t/S_0) \sim N\left((\mu - \frac{\sigma^2}{2})t, \sigma^2 t\right)$$

> 🔑 **关键洞察**：为什么对数收益率均值是 (μ - σ²/2)t 而不是 μt？
> → 这是伊藤积分的凸性调整项，来自 d(ln S) 展开时 σ²S² 的二阶项不消失（因为 (dW_t)² ≈ dt）

**鞅（Martingale）概念**：

在风险中性测度下，折现价格过程是鞅：E[S_T / e^{rT} | F_t] = S_t / e^{rt}

### 🐍 Python 实战：GBM 模拟与期权定价

```python
"""
随机过程与 Black-Scholes（资深进阶）
"""

import numpy as np
from scipy import stats
from typing import Tuple


# ============ GBM 路径生成 ============

def simulate_gbm_paths(
    S0: float = 100,
    mu: float = 0.10,          # 年化漂移
    sigma: float = 0.20,       # 年化波动率
    T: float = 1.0,            # 期限（年）
    n_steps: int = 252,        # 时间步数
    n_paths: int = 10000,      # 路径数
    seed: int = 42
) -> np.ndarray:
    """
    生成 GBM 样本路径

    精确离散化：
    S_{t+Δt} = S_t · exp((μ - σ²/2)Δt + σ√Δt · ε_t)

    注意：这里用的是精确解，不是 Euler 近似。
    Euler 近似在 Δt 小时才准确，而精确离散化在任意 Δt 下都准确。
    """
    rng = np.random.default_rng(seed)
    dt = T / n_steps
    drift = (mu - 0.5 * sigma**2) * dt
    diffusion = sigma * np.sqrt(dt)

    # 生成所有随机数（矩阵形式，内存高效）
    z = rng.normal(0, 1, (n_steps, n_paths))

    # 累积对数收益率 → 指数 → 得到价格路径
    log_returns = drift + diffusion * z
    log_prices = np.log(S0) + np.cumsum(log_returns, axis=0)

    # 在开头插入 S0
    prices = np.exp(log_prices)
    prices = np.vstack([np.full((1, n_paths), S0), prices])

    return prices  # (n_steps+1, n_paths)


# ============ Black-Scholes 期权定价 ============

def black_scholes(
    S: float,        # 标的价格
    K: float,        # 行权价
    T: float,        # 剩余期限（年）
    r: float,        # 无风险利率
    sigma: float,    # 波动率
    option_type: str = "call"
) -> float:
    """
    Black-Scholes 公式

    推导思路（面试中可能被要求阐述）：
    1. 构造无风险组合：卖 1 份期权 + 买 Δ 份标的
    2. 应用伊藤引理，消除随机项 dW_t
    3. 得到 BS 偏微分方程：∂V/∂t + rS ∂V/∂S + ½σ²S² ∂²V/∂S² = rV
    4. 用热传导方程求解（或等价鞅测度法）

    Call:  C = S·N(d1) - K·e^{-rT}·N(d2)
    Put:   P = K·e^{-rT}·N(-d2) - S·N(-d1)

    d1 = [ln(S/K) + (r + σ²/2)T] / (σ√T)
    d2 = d1 - σ√T

    重要性质（面试常考）：
    - N(d2) = 在风险中性测度下，期权被行权的概率
    - N(d1) = Delta (call) = 每单位标的价格变动时期权价格的变动
    """
    if T <= 0:
        # 到期日：内在价值
        if option_type == "call":
            return max(S - K, 0)
        return max(K - S, 0)

    d1 = (np.log(S / K) + (r + 0.5 * sigma**2) * T) / (sigma * np.sqrt(T))
    d2 = d1 - sigma * np.sqrt(T)

    if option_type == "call":
        return float(S * stats.norm.cdf(d1) - K * np.exp(-r * T) * stats.norm.cdf(d2))
    else:
        return float(K * np.exp(-r * T) * stats.norm.cdf(-d2) - S * stats.norm.cdf(-d1))


def black_scholes_greeks(
    S: float, K: float, T: float, r: float, sigma: float
) -> dict:
    """
    Black-Scholes Greeks（风险敏感度）

    面试中至少要知道 Delta, Gamma, Theta, Vega 的含义：
    - Delta (Δ):  期权价格对标的价格的敏感度 ∂V/∂S
    - Gamma (Γ):  Delta 对标的价格的敏感度 ∂²V/∂S²（凸性）
    - Theta (Θ):  期权价格对时间的敏感度 ∂V/∂t（时间衰减）
    - Vega (ν):   期权价格对波动率的敏感度 ∂V/∂σ
    - Rho (ρ):    期权价格对利率的敏感度 ∂V/∂r（通常影响很小）
    """
    d1 = (np.log(S / K) + (r + 0.5 * sigma**2) * T) / (sigma * np.sqrt(T))
    d2 = d1 - sigma * np.sqrt(T)
    sqrt_T = np.sqrt(T)

    # Call Greeks
    delta = float(stats.norm.cdf(d1))
    gamma = float(stats.norm.pdf(d1) / (S * sigma * sqrt_T))
    theta = float(
        -S * stats.norm.pdf(d1) * sigma / (2 * sqrt_T)
        - r * K * np.exp(-r * T) * stats.norm.cdf(d2)
    )
    vega = float(S * sqrt_T * stats.norm.pdf(d1) / 100)  # 除以 100 → 1% 波动率变化
    rho = float(K * T * np.exp(-r * T) * stats.norm.cdf(d2) / 100)

    return {
        "price": black_scholes(S, K, T, r, sigma, "call"),
        "delta": delta,
        "gamma": gamma,
        "theta": theta / 252,  # Theta per day
        "vega": vega,
        "rho": rho,
    }


# ============ 演示 ============

if __name__ == "__main__":
    # GBM 模拟
    prices = simulate_gbm_paths(S0=100, mu=0.10, sigma=0.20,
                                 T=1.0, n_steps=252, n_paths=10000)

    final_prices = prices[-1, :]
    log_returns = np.log(final_prices / 100)
    actual_mu = np.mean(log_returns)
    actual_sigma = np.std(log_returns)

    print("=" * 60)
    print("  GBM 模拟验证")
    print("=" * 60)
    print(f"  对数收益率均值: {actual_mu:.4f} "
          f"(理论: (μ-σ²/2)T = {(0.10 - 0.5*0.04)*1:.4f})")
    print(f"  对数收益率标准差: {actual_sigma:.4f} "
          f"(理论: σ√T = {0.20:.4f})")
    print(f"  S_T 均值: {np.mean(final_prices):.2f} "
          f"(理论: S₀·e^{μT} = {100*np.exp(0.10):.2f})")

    # Black-Scholes
    print(f"\n{'='*60}")
    print("  Black-Scholes 期权定价")
    print("=" * 60)
    greeks = black_scholes_greeks(
        S=100, K=100, T=30/365, r=0.05, sigma=0.20
    )
    for greek, val in greeks.items():
        print(f"  {greek}: {val:.6f}")

    # 波动率微笑的暗示
    print(f"\n  ⚠️  资深知识点：BS 假设波动率恒定，")
    print(f"     但实际市场存在 volatility smile/skew。")
    print(f"     这意味着 BS 模型对深度虚值/实值期权定价不准确。")
```

```python
def is_power_of_two(n: int) -> bool:
    """
    判断 n 是否为 2 的幂次方（1, 2, 4, 8, 16, ...）

    核心技巧：2 的幂次方的二进制表示只有 1 个 '1'
    n & (n-1) 会消去最低位的 1

    例：
      n=8  (1000), n-1=7  (0111), n & (n-1) = 0000 = 0  ✅
      n=10 (1010), n-1=9  (1001), n & (n-1) = 1000 ≠ 0  ❌

    时间复杂度：O(1)
    空间复杂度：O(1)
    """
    # 注意边界：n=0 时 n-1=-1，位运算是未定义的，所以要 n>0
    return n > 0 and (n & (n - 1)) == 0


# 测试
for test_n in [1, 2, 3, 4, 8, 10, 16, 256]:
    print(f"  is_power_of_two({test_n:>3}) = {is_power_of_two(test_n)}")
```

**题26**：蓄水池抽样——从长度为 N 的流中随机抽取 k 个元素，N 未知。

```python
import random
from typing import List, Iterator, TypeVar

T = TypeVar('T')


def reservoir_sampling(stream: Iterator[T], k: int) -> List[T]:
    """
    蓄水池抽样（Reservoir Sampling）

    问题：从长度为 N（未知）的数据流中，等概率地随机抽取 k 个元素。
    关键约束：只能遍历一次，不能存储所有元素。

    算法：
    1. 前 k 个元素直接放入蓄水池
    2. 对于第 i 个元素 (i > k)，以 k/i 的概率替换蓄水池中随机一个元素

    证明（数学归纳法）：
    任意元素 X_j 最终留在蓄水池中的概率 = k/N

    时间：O(N)
    空间：O(k)

    面试高频题：LeetCode #382, #398
    """
    reservoir: List[T] = []

    for i, item in enumerate(stream):
        if i < k:
            # 前 k 个元素直接放入
            reservoir.append(item)
        else:
            # 以 k/(i+1) 的概率替换蓄水池中的随机一个元素
            j = random.randint(0, i)  # [0, i] 均匀随机
            if j < k:
                reservoir[j] = item

    return reservoir


# 验证：模拟 10000 次，检查每个元素被选中的概率是否相等
def verify_reservoir():
    """蒙特卡洛验证蓄水池抽样的均匀性"""
    n = 10   # 流中总元素数
    k = 3    # 抽样数
    trials = 20000

    # 统计每个元素出现在结果中的次数
    count = {i: 0 for i in range(n)}

    for _ in range(trials):
        result = reservoir_sampling(iter(range(n)), k)
        for item in result:
            count[item] += 1

    expected = trials * k / n  # 期望出现次数
    print(f"\n蓄水池抽样验证 (n={n}, k={k}, trials={trials})：")
    print(f"  期望每个元素出现次数: {expected:.0f}")
    for i in range(n):
        print(f"  元素 {i}: {count[i]} 次 "
              f"({count[i]/expected:.3f}x 期望)")


verify_reservoir()
```

**题27**：判断一组数字能否构成等差数列。

```python
def can_make_arithmetic_progression(arr: List[int]) -> bool:
    """
    判断数组重新排列后能否构成等差数列

    方法1（推荐，O(nlogn)）：排序后检查相邻差
    方法2（O(n)）：找到 min 和 max，利用等差性质验证

    LeetCode #1502
    """
    if len(arr) <= 2:
        return True

    # --- 方法1：排序法 ---
    arr_sorted = sorted(arr)
    diff = arr_sorted[1] - arr_sorted[0]
    for i in range(2, len(arr_sorted)):
        if arr_sorted[i] - arr_sorted[i - 1] != diff:
            return False
    return True


def can_make_arithmetic_progression_o_n(arr: List[int]) -> bool:
    """
    方法2（O(n) 时间，O(n) 空间）：

    利用等差数列性质：min 和 max 确定后，
    所有元素都可以通过 min + k*d 计算出来
    """
    n = len(arr)
    if n <= 2:
        return True

    min_val = min(arr)
    max_val = max(arr)

    # 公差必须是整数
    if (max_val - min_val) % (n - 1) != 0:
        return False

    d = (max_val - min_val) // (n - 1)
    if d == 0:
        # 所有元素相等
        return len(set(arr)) == 1

    # 用集合检查每个元素是否在期望位置
    expected = set(min_val + i * d for i in range(n))
    return set(arr) == expected


# 测试
test_cases = [
    [3, 5, 1],           # True: 1, 3, 5
    [1, 2, 4],           # False
    [0, 0, 0],           # True: d=0
    [1, 3, 5, 7, 9],    # True
]
for arr in test_cases:
    print(f"  {arr} → {can_make_arithmetic_progression(arr)}")
```

**题28**（额外高频）：用 Rand7() 实现 Rand10()。

```python
import random


def rand7() -> int:
    """给定 API：返回 1~7 的均匀随机整数"""
    return random.randint(1, 7)


def rand10() -> int:
    """
    用 Rand7() 实现 Rand10() —— 拒绝采样（Rejection Sampling）

    面试高频题！LeetCode #470

    核心思路：
    1. 调用两次 rand7()，构造 7×7=49 种等可能结果
    2. 映射到 1~40（取前 40 个），取模得到 1~10
    3. 41~49 的情况拒绝，重新采样

    期望调用次数：49/40 × 2 ≈ 2.45 次 rand7()
    """
    while True:
        # (rand7()-1)*7 + (rand7()-1) → 0~48 共 49 种等可能结果
        row = rand7() - 1        # 0~6
        col = rand7() - 1        # 0~6
        idx = row * 7 + col      # 0~48 均匀分布

        # 拒绝 40~48（即 idx >= 40 时重新采样）
        if idx < 40:
            # 映射到 1~10
            return (idx % 10) + 1


# 验证均匀性
def verify_rand10(trials: int = 100000):
    """蒙特卡洛验证 rand10() 的均匀性"""
    count = {i: 0 for i in range(1, 11)}
    for _ in range(trials):
        count[rand10()] += 1

    print(f"\nRand10() 均匀性验证 ({trials} 次采样)：")
    expected = trials / 10
    for i in range(1, 11):
        print(f"  值 {i:>2}: {count[i]:>6} 次 "
              f"({count[i]/expected:.3f}x 期望)")
    print(f"  卡方近似: "
          f"{sum((c - expected)**2 / expected for c in count.values()):.2f}")


verify_rand10(50000)
```

## 面试技巧（分级别）

### 🟢 初级岗面试策略

1. **展示推理过程**：量化面试中，推理过程比正确答案更重要。边想边说。
2. **明确假设**：涉及概率时，先说清楚你的假设（独立？均匀分布？）。
3. **估算能力**：练习心算和数量级估算，很多题目不要求精确值。
4. **连接金融场景**：尽量把数学概念和交易/投资场景联系起来。
5. **问清楚条件**：博弈题先搞清规则（是否同时出招、信息是否对称等）。

### 🔥 资深岗面试策略

1. **主动指出假设的局限**：当你说"假设正态分布"时，立刻补充"但实际收益率有肥尾特征，所以这只是一个近似，更准确的做法是..."。面试官会因此加分。

2. **展示数值直觉**：
   - 夏普 1.0 是"还不错"，2.0 是"很优秀"，0.5 以下"不太行"
   - 年化波动率 15% 是大盘股水平，30% 是小盘/科技股，50%+ 是个股或加密
   - MaxDD 超过 2× 年化波动率说明存在肥尾或一连串坏事件

3. **提到实际工作的复杂性**：
   - "我在这个推导中用样本协方差矩阵，但实践中我会用 Ledoit-Wolf shrinkage estimator，因为样本协方差在 N/T 比例较大时非常不稳定"
   - "夏普比率的 √252 年化法则假设收益率 i.i.d.，如果存在自相关需要用 Newey-West 修正标准误"
   - "K 层推理博弈的收敛速度取决于参与者的认知分布，这类似于预测市场时要判断'有多少人比我想得更远'"

4. **用代码辅助推理**：面试中说"让我用代码验证一下这个结论"是资深工程师的思维习惯，展现了严谨性。

5. **知道每个概念的 trade-off**：
   - 全凯利 vs 半凯利 → 增长 vs 回撤
   - Long-only vs 允许做空 → 稳定性 vs 灵活性
   - 参数法 VaR vs 历史模拟法 → 效率 vs 稳健性
   - 高频因子 vs 低频因子 → 容量 vs 信噪比

6. **准备"之前的项目"的深度追问**：如果简历上写了回测系统或因子研究，准备好被问到：
   - 怎么处理生存偏差（survivorship bias）？
   - 怎么处理前视偏差（look-ahead bias）？
   - 你做的回测中，最大的陷阱是什么？
   - 你怎么区分过拟合和真正的 alpha？
