# AI/全栈/后端岗位 通用技术栈总结
以下是从所有JD中提炼出的、跨Go/Python等语言通用的后端技术栈与能力要求，按模块整理：

---

## 一、后端核心开发能力
- **编程语言**：至少精通一种主流后端语言（Python/Go/Java/Node.js 等），具备扎实的工程基础
- **Web 框架**：FastAPI、Express、Spring Boot 等，能独立构建高性能 API 服务
- **API 设计**：RESTful API / GraphQL 设计与实现
- **高并发与分布式**：高并发系统设计与优化、分布式架构模式、一致性与可用性权衡、限流熔断、灰度发布、稳定性治理

---

## 二、数据与存储技术
- **关系型数据库**：MySQL、PostgreSQL
- **NoSQL 数据库**：MongoDB、Redis（缓存机制、分布式缓存）
- **向量数据库**：Milvus、ES、Qdrant（AI 应用必备）
- **大数据技术栈（加分项）**：MaxCompute、Holo、StarRocks、Flink

---

## 三、消息与异步处理
- **消息中间件**：Kafka、RabbitMQ
- **异步任务队列**：Celery、BullMQ
- **事件驱动架构**：消息队列/事件驱动、任务编排（状态机、DAG、幂等、重试、超时、补偿、可回放、任务队列隔离）

---

## 四、实时通信与服务治理
- **实时通信**：WebSocket / Socket.IO 等技术实现
- **服务治理**：服务化部署、弹性扩缩容、可观测性（监控告警、日志追踪）、故障演练
- **容器化与编排**：Docker、Kubernetes（含 K3s/边缘集群经验更佳）

---

## 五、DevOps 与工程化能力
- **环境与工具**：Linux 基础操作、Git 版本管理、CI/CD 流程
- **云服务**：AWS/GCP/Azure 等主流云平台使用
- **工程规范**：良好的编码习惯、代码可维护性、自动化校验、测试与部署流程

---

## 六、AI 应用工程化核心能力（所有岗位高频提及）
- **LLM 应用开发**：大模型调用方式（OpenAI/Anthropic 等）、LLM 应用从 0 到 1 上线与迭代经验
- **关键技术栈**：
  - RAG（检索增强生成）与知识库搭建
  - Agent 开发框架（LangChain、Dify、Context Engineering 等）
  - 工具调用/插件化框架（Function Calling、MCP、工具注册-鉴权-执行-审计平台）
  - Workflow/任务编排、记忆/上下文治理、提示工程、结构化输出
- **效果评估与优化**：
  - 指标体系：准确率、召回率、相关性、任务完成率、用户采纳率、幻觉率、响应时延、单位调用成本
  - 评测方法：Prompt 评测、RAG 评测、Agent 任务成功率评测、人工评审标准设计、A/B 实验、数据闭环系统
- **前沿技术（加分项）**：Browser Use/Computer Use GUI 自动化交互、本体论与知识图谱、多智能体协作、Code Interpreter 沙箱执行与安全隔离

---

## 七、通用软技能与加分项
- 独立问题拆解能力、模糊需求处理能力
- 跨团队沟通协作能力、产品意识与业务视角
- 学习能力与技术跟进能力，能快速跟进 AI 技术、模型能力和工具链变化
- 加分项：
  - 广告/搜索/推荐系统相关经验（召回、排序、粗排、精排、特征工程、用户画像）
  - 金融/大数据领域 Agent 开发经验
  - 开源项目、技术博客、AI 产品作品或独立开发项目经验
  - 大规模撮合系统、业务系统经验
  - AI 编程工具使用经验（Cursor、Claude Code、GitHub Copilot 等），能区分工具适用场景并进行修正

---

## 补充说明
1.  所有岗位的核心后端能力（并发、分布式、存储、消息队列、容器化）是通用的，与具体语言无关；
2.  AI 相关岗位额外强调了 LLM/Agent/RAG 等应用层工程化能力，以及效果评估、成本优化、安全隔离等实践经验；
3.  软技能中，“业务视角+问题拆解+跨团队协作”是大厂岗位普遍看重的加分项。
