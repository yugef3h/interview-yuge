## 一、业务与 AI Agent 相关
1.  实习拷打（10 min）
2.  介绍你做过的 Agent 项目整体流程
    - 为什么这么拆流程？
    - 哪些节点是确定性 Workflow，哪些节点交给 LLM 决策？
    - 有没有失败分支和重试机制？
    - 状态是怎么保存和恢复的？
    - 多 Agent 如何分工、通信和终止？
    - 如何避免多个 Agent 相互扯皮或死循环？
3.  如何设计 Tool Calling
    - Tool 的输入输出 schema 怎么定义？
    - 工具调用失败怎么处理？
    - 工具参数错误怎么兜底？
    - 如何防止模型调用危险工具？
    - Tool 是直接暴露给模型，还是通过服务端做分发？
    - 如何判断是 Prompt 问题还是模型能力问题？
4.  项目中遇到的模型不听指令问题
    - 具体 bad case 是什么？
    - 是怎么定位的？
    - 通过 Prompt、模型参数、后处理还是流程约束解决？
    - 解决后有没有评测数据证明有效？
    - 反思结果会不会污染上下文？
5.  Claude Code 的 Memory 机制
    - 为什么要分层记忆？
    - 项目级规则、用户偏好、会话上下文应该怎么隔离？
    - 上下文过长时性能下降怎么处理？
6.  RAG 优化相关
    - 为什么要加 Rerank？
    - Recall@K 和 Precision@K 怎么取舍？
    - Top-K 怎么动态调整？
    - 如果 BM25 已经很好，向量检索还有必要吗？
7.  Agentic RAG 与传统 RAG 的区别
    - Query Rewrite 是否算 Agentic？
    - 多轮检索如何设计？
    - 检索失败后 Agent 应该怎么调整策略？
    - Agentic RAG 的成本和稳定性问题怎么控制？
8.  使用过的 Agent 框架
    - LangGraph 和 LangChain 的区别是什么？
    - LangGraph 里的 State、Node、Edge 分别解决什么问题？
    - 为什么选择 LangGraph，而不是 AutoGen、CrewAI 或手写状态机？
    - 框架能力不满足时你怎么扩展？
9.  对扣子这类低代码 Agent 平台的看法
    - 低代码 Agent 平台的优势和局限是什么？
    - 和 LangGraph 这类代码框架怎么选？
10. 你平时怎么使用 AI 编程工具
    - 使用的大概流程是什么样子？
    - 如何保证 AI 生成代码质量？
    - 有没有规则文件、测试、Code Review 兜底？

## 二、计算机基础 / 八股
1.  Redis 常见数据结构及场景
    - String、Hash、List、Set、ZSet 分别怎么用？
    - Agent 系统里的会话状态适合用哪种结构存？
2.  Redis 的过期删除策略和内存淘汰策略
    - Redis Key 过期后会立刻删除吗？
    - 惰性删除和定期删除是什么？
    - maxmemory-policy 有哪些？
    - 缓存会话状态时应该选什么淘汰策略？
3.  缓存穿透、缓存击穿、缓存雪崩
    - 缓存穿透怎么解决？
    - 布隆过滤器有什么作用？
    - 缓存击穿为什么要加互斥锁？
    - 缓存雪崩为什么要设置随机过期时间？
4.  Redis Stream 和 Kafka 的区别
    - Redis Stream 是否能保证消息不丢？
    - Pending List 是什么？
    - 消息确认 ACK 怎么做？
    - 什么情况下用 Redis Stream，什么情况下用 Kafka？
5.  Transformer 基础
    - Self-Attention 是怎么计算的？
    - Multi-Head Attention 的作用是什么？
    - GPT / BERT 的结构差异是什么？
6.  SFT、RLHF、DPO 的区别
    - 为什么 SFT 后还需要偏好优化？
